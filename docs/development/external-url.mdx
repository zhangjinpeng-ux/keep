## 简介
Keep 中的多个功能需要一个可从互联网访问的外部 URL。这对于安装提供商时的 Webhook 集成等功能尤为重要。Keep 在此过程中使用其 API URL 将自身建立为 Webhook 连接器。

当告警被触发时，相应的提供商尝试激活 webhook，传递告警负载。因此，webhook 必须可以通过互联网访问，此过程才能有效工作。

## 使用 NGROK 实现外部可访问性


Keep 支持使用 NGROK 创建可访问的外部 URL。通过使用环境变量 USE_NGROK=true 启动 Keep，Keep 将自动启动 NGROK 隧道并利用此 URL 进行 webhook 安装。

<Note>
  虽然 `USE_NGROK` 在开发或测试时很方便，但需要注意的是，每次重启 Keep 都会生成一个新的 NGROK URL。URL 的这种变化意味着使用旧 URL 配置的提供商将无法再与 Keep 通信。


  对于生产环境，建议：
  - 使用永久的、可通过互联网访问的 URL 暴露 Keep。
  - 设置静态 NGROK 隧道。

  随后，通过设置 KEEP_API_URL 环境变量将 Keep 配置为使用此稳定的 URL。

</Note>
