---
title: "使用 GitHub Action 同步 Keep 工作流"
---

本文档详细介绍了如何在 GitHub Actions 工作流中使用 Keep CLI 来同步和管理目录中的 Keep 工作流。此设置可自动执行将工作流上传到 Keep 的过程，使其更易于维护和更新。





### 配置
要在你的仓库中设置此工作流：

- 将工作流 YAML 文件添加到仓库的 `.github/workflows/` 目录下。
- 如果尚未设置，请在仓库设置中将你的 Keep API Key 和 URL 设置为 secrets。
- 对指定目录中的工作流进行更改，或通过 GitHub UI 手动触发工作流。
- 将 'example/workflows/**' 更改为你存储 Keep 工作流的目录。


### GitHub Action 工作流
此 GitHub Actions 工作流会在有更改时自动将指定目录中的工作流同步到 Keep。它还支持通过可选参数手动触发。

```yaml
# 一个同步 Keep 工作流的 workflow
name: "Sync Keep Workflows"

on:
    push:
        paths:
          - 'examples/workflows/**'
    workflow_dispatch:
        inputs:
            keep_api_key:
              description: 'Keep API Key'
              required: false
            keep_api_url:
              description: 'Keep API URL'
              required: false
              default: 'https://api.keephq.dev'

jobs:
    sync-workflows:
        name: Sync workflows to Keep
        runs-on: ubuntu-latest
        container:
            image: us-central1-docker.pkg.dev/keephq/keep/keep-cli:latest
        env:
            KEEP_API_KEY: ${{ secrets.KEEP_API_KEY || github.event.inputs.keep_api_key }}
            KEEP_API_URL: ${{ secrets.KEEP_API_URL || github.event.inputs.keep_api_url }}

        steps:
        - name: Check out the repo
          uses: actions/checkout@v2

        - name: Run Keep CLI
          run: |
            keep workflow apply -f examples/workflows

```
