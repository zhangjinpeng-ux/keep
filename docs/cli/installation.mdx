<Info>
我们建议使用 Python 3.11 版本来安装 Keep CLI，以获得最佳的兼容性和性能。
此选择可确保与所有依赖项（包括 pyarrow）的无缝集成，pyarrow 目前不支持 Python 3.12。
</Info>

<Tip>需要其他版本的 Keep CLI？欢迎联系我们！</Tip>

## 克隆并安装（方案一）

### 安装
首先，克隆 Keep 仓库：

```shell
git clone https://github.com/keephq/keep.git && cd keep
```

使用 `pip` 安装 Keep CLI：

```shell
# MacOS 如果未安装 python 或 pip：
# brew install python@3.11
# brew install postgresql
pip3.11 install .
```
或使用 `poetry`：

```shell
poetry install
```

从现在起，Keep 应该已本地安装并可从你的 CLI 访问，通过执行以下命令进行测试：

```
keep version
```

### 配置

要获取 API key，请查看 Keep UI -> 你的用户名（左下角）-> Settings -> API Keys
```
keep config new --url http://backend.my_keep.my_awesome_org.com:backend_port --api-key your_personal_api_key
```

### 测试

现在，
```
keep workflow apply -f examples/workflows/query_clickhouse.yml
```

恭喜 🥳 检查你的 UI，查看从 YAML 文件上传的新工作流。


## Docker 镜像（方案二）
### 安装

```
docker run -v ${PWD}:/app -v ~/.keep.yaml:/root/.keep.yaml -it us-central1-docker.pkg.dev/keephq/keep/keep-cli keep config new --url http://backend.my_keep.my_awesome_org.com:backend_port --api-key your_personal_api_key
```

### 测试
```
docker run -v ${PWD}:/app -v ~/.keep.yaml:/root/.keep.yaml -it us-central1-docker.pkg.dev/keephq/keep/keep-cli workflow apply -f examples/workflows/query_clickhouse.yml
```


## 启用自动补全
Keep CLI 支持 shell 自动补全，这可以让你的操作更加便捷 😌
如果你使用 zsh：

```shell title=~/.zshrc
eval "$(_KEEP_COMPLETE=zsh_source keep)"
```

如果你使用 bash：

```bash title=~/.bashrc
eval "$(_KEEP_COMPLETE=bash_source keep)"
```

<Info>使用 eval 意味着每次启动 shell 时都会调用并执行该命令，这可能会影响 shell 的响应速度。要加快速度，可以将生成的脚本写入文件，然后 source 该文件。</Info>
