Keep 的告警提取富化功能支持使用正则表达式从传入告警中动态提取数据。这个强大的工具允许用户定义提取规则，基于模式识别和提取数据，用直接从告警内容中提取的额外结构化数据来富化告警。

## 简介

处理各种告警格式并提取相关信息可能具有挑战性。Keep 的告警提取功能通过允许用户定义基于正则表达式的规则来自动从告警中提取关键信息，简化了这一过程。此功能对于标准化告警数据和增强告警上下文至关重要，有助于更有效的监控和响应策略。

## 工作原理

1. **规则定义**：用户创建提取规则，指定要应用于某些告警属性的正则表达式模式。
2. **属性指定**：每个规则定义告警的哪个属性应由正则表达式检查。
3. **数据提取**：当收到告警时，系统将正则表达式应用于指定的属性。如果模式匹配，正则表达式中的命名组定义要提取并添加到告警中的新属性。
4. **首次匹配强制执行**：提取过程设计为在第一次成功匹配后停止。一旦规则成功应用并富化了告警，就不再处理其他规则。这确保了效率并防止重叠或冗余的数据提取。
5. **告警富化**：提取的值被添加到告警中，用额外的属性增强其数据以改善分析。

## 实际示例

假设您收到的告警具有格式为 "Error 404: Not Found - [UserID: 12345]" 的消息属性。您可以定义一个提取规则，使用正则表达式如 `Error (?P<error_code>\d+): (?P<error_message>.+) - \[UserID: (?P<user_id>\d+)\]` 将 `error_code`、`error_message` 和 `user_id` 提取为告警中的独立属性。

## 核心概念

- **Regex（正则表达式）**：一种强大的模式匹配语法，用于识别文本中的特定模式。在提取规则的上下文中，regex 用于定义如何从告警属性中提取数据。正则表达式模式必须遵循 [Python 的正则表达式语法](https://docs.python.org/3.11/library/re.html#match-objects)，特别是关于使用命名组进行组匹配的部分。
- **属性**：正则表达式应用于的告警数据部分（例如 message、description）。
- **命名组**：正则表达式模式的一部分，指定用于将特定数据点提取到新告警属性中的占位符。

## 创建提取规则

创建告警提取规则：

1. **选择属性**：选择告警的哪个属性应由正则表达式检查。
2. **定义正则表达式**：编写带有命名组的正则表达式模式，指定要提取的信息。确保正则表达式根据 Python 的正则表达式标准有效，特别是对于组匹配。
3. **配置条件**：可选地，使用 CEL（Common Expression Language）指定此规则应适用的条件，以支持复杂逻辑。

## 最佳实践

- **测试正则表达式模式**：在部署新的提取规则之前，彻底测试正则表达式模式，确保其根据 Python 的正则表达式标准正确匹配和提取数据。
- **监控提取性能**：跟踪提取规则的表现以及它们是否按预期富化告警。根据传入的告警数据根据需要调整模式。
- **使用具体条件**：在适用时定义条件以限制提取规则的应用范围，减少不必要的处理并专注于相关告警。
