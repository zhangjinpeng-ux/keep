Keep 使用 **CEL（Common Expression Language）** 作为一个强大而灵活的工具，用于根据预定义规则评估和过滤告警。CEL 使用户能够编写精确的表达式，定义告警被处理、显示或执行操作的条件。此功能通过允许对传入告警的可见性和响应进行细粒度控制，增强了告警管理能力。

## Keep 如何使用 CEL

### 告警过滤
告警会根据 CEL 表达式动态评估，以确定哪些告警满足指定的条件。这种实时过滤确保只有最相关的告警才会被呈现。

### 规则评估
CEL 表达式可以嵌入到规则中，以强制执行特定操作，例如升级告警或触发工作流。

### 预设
用户可以将常用的 CEL 表达式保存为预设，以便在不同的告警视图或团队中快速且一致地应用。

## 示例

### 过滤来自特定服务的告警

```cel
service.contains("database")
```

### 组合多个条件

```cel
severity == "critical" && source == "prometheus"
```

### 排除特定告警

```cel
!(service == "auth" && severity == "low")
```
