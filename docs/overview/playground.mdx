使用 Keep 的 [playground](https://playground.keephq.dev) 来探索、实验并了解 Keep 如何简化运维和减少噪音，使您能够清晰地掌控 IT 生态系统。

## 告警

通过可自定义的预设，获得所有告警的统一视图。使用 CEL（Common Expression Language）语法进行精确过滤，配置告警表格布局以匹配您的工作流程，并通过多维度分析探索告警模式和指标的快速洞察。

## 事件

详细检查事件，包括其关联的告警和时间线。测试将相关告警分组为事件的关联逻辑和映射配置，并验证您的抑制或解决策略。

## 提供商

与外部数据源或告警提供商（如 Prometheus、Datadog 或 GCP Monitoring）集成。配置和测试映射，确保从各种来源正确摄入和规范化数据到 Keep 的统一模式中。

## 工作流

构建和测试自动化工作流，精确管理告警和事件。使用直观的 UI 构建器和高级脚本功能进行实验，根据动态条件触发操作、通知或外部集成。

## AIOps 技术

测试和完善去重、富化、映射和提取规则，优化告警处理。通过这些技术进行实验，将原始告警转化为可操作的数据，有效减少噪音。
