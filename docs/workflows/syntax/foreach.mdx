---
title: "Foreach"
---

Keep 工作流中的 `foreach` 指令允许您遍历项目列表，并对每个项目执行操作。这对于处理步骤返回的多个结果或对一组实体执行操作特别有用。

## 核心特性

- **动态迭代：** 遍历步骤返回或工作流中定义的任意列表或数组。
- **作用域变量：** 每次迭代将当前项目暴露在 `foreach` 变量下，允许您直接访问其属性。
- **动作链式调用：** 多个动作可以使用 `foreach` 对同一个项目列表顺序执行。

---

## 定义 `foreach`

要使用 `foreach`，请将其作为动作的一部分。`foreach` 的值应为要遍历的列表的引用。

### 使用 `foreach` 的工作流示例

```yaml
workflow:
  id: foreach-example
  description: Demonstrates the use of foreach
  triggers:
    - type: manual
  steps:
    - name: get-pods
      provider:
        type: gke
        config: "{{ providers.gke }}"
        with:
          command_type: get_pods
  actions:
    - name: echo-pod-status
      foreach: "{{ steps.get-pods.results }}"
      provider:
        type: console
        with:
          message: "Pod name: {{ foreach.value.metadata.name }} || Namespace: {{ foreach.value.metadata.namespace }} || Status: {{ foreach.value.status.phase }}"
```

在此示例中：

- `get-pods` 步骤获取 Kubernetes pods 列表。
- `foreach` 遍历 `get-pods` 步骤返回的 `results`。
- 对于每个 pod，打印其 `name`、`namespace` 和 `status`。

---


## 使用 `foreach` 变量

`foreach` 变量为当前迭代中的项目提供作用域内的访问。

### 作用域变量示例

```yaml
actions:
  - name: notify-pod-status
    foreach: "{{ steps.get-pods.results }}"
    provider:
      type: slack
      with:
        message: |
          Pod Name: {{ foreach.value.metadata.name }}
          Namespace: {{ foreach.value.metadata.namespace }}
          Status: {{ foreach.value.status.phase }}

```

在此情况下：
- `{{ foreach.value }}` 引用列表中的当前项目。
- 动态访问 `metadata.name`、`metadata.namespace` 和 `status.phase` 等属性。


### 将条件与 `foreach` 结合使用

您可以将 `foreach` 与 `if` 条件结合使用，以进行过滤或选择性操作。

```yaml
actions:
  - name: alert-critical-pods
    foreach: "{{ steps.get-pods.results }}"
    if: "{{ foreach.value.status.phase == 'Failed' }}"
    provider:
      type: slack
      with:
        message: "Critical pod failure detected: {{ foreach.value.metadata.name }}"
```
