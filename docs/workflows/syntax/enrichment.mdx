---
title: "增强"
---

Keep 工作流支持**增强（Enrichment）**，这是一项强大的功能，允许您为告警添加额外数据，使其更具可操作性和意义。增强功能可直接从工作流中向告警添加自定义字段或修改现有字段。

---

## 为什么要增强告警？

- **提供上下文：** 添加关键信息，例如相关客户数据或工单 ID。
- **启用自动化：** 在后续动作中使用增强后的字段进行动态处理。
- **提高可见性：** 展示必要的元数据以辅助更好的决策。

---

## 如何增强告警

### 使用 `enrich_alert` 指令

`enrich_alert` 指令在动作中用于添加或更新告警中的字段。您需要指定一组键值对，其中：
- `key` 是要添加或更新的字段名。
- `value` 是要分配给该字段的数据。可以是静态值，也可以是从步骤或工作流其他部分动态派生的值。
- `disposable` 是一个可选属性，用于确定增强是否为临时性的，并在收到新告警时丢弃。如果 disposable 设置为 True，增强内容将被添加到 disposable_enrichments 中，并标记为 dispose_on_new_alert=True。

### 带增强的工作流示例

```yaml
workflow:
  id: enrich-alert-example
  description: Demonstrates enriching alerts
  triggers:
    - type: alert
  steps:
    - name: get-customer-details
      provider:
        type: mysql
        config: "{{ providers.mysql-prod }}"
        with:
          query: "SELECT * FROM customers WHERE customer_id = '{{ alert.customer_id }}'"
          single_row: true
  actions:
    - name: enrich-alert-with-customer-data
      provider:
        type: mock
        with:
          enrich_alert:
            - key: customer_name
              value: "{{ steps.get-customer-details.results.name }}"
            - key: customer_tier
              value: "{{ steps.get-customer-details.results.tier }}"
```

在此示例中：
- `get-customer-details` 步骤根据告警获取客户数据。
- `enrich_alert` 指令将 `customer_name` 和 `customer_tier` 添加到告警中。

## 增强语法

### 键值对
每个增强定义为一个键值对：

```yaml
enrich_alert:
  - key: field_name
    value: field_value
    disposable: true
```

- **静态值：** 使用静态字符串或数字进行简单的增强：
```yaml
- key: alert_source
  value: "Monitoring System"
```

-- **动态值：** 使用从步骤、动作或告警本身派生的值：
```yaml
- key: severity_level
  value: "{{ alert.severity }}"
```

### 条件增强

您可以将增强与条件结合使用，以动态增强告警：

```yaml
actions:
  - name: enrich-critical-alert
    if: "{{ alert.severity == 'critical' }}"
    provider:
      type: mock
      with:
        enrich_alert:
          - key: priority
            value: high
```

## 高级用例


### 使用动作结果增强告警
增强可以使用动作的结果，从而基于前序步骤进行动态更新：
```yaml
enrich_alert:
  - key: ticket_id
    value: "{{ actions.create-ticket.results.ticket_id }}"
  - key: ticket_url
    value: "{{ actions.create-ticket.results.ticket_url }}"

```

## 增强工作流示例

此示例演示了如何使用 ServiceNow 的工单详情来增强告警：

```yaml
workflow:
  id: servicenow-ticket-enrichment
  triggers:
    - type: alert
  steps:
    - name: fetch-alert-details
      provider:
        type: keep
        with:
          filter: "alert_id == '{{ alert.id }}'"
  actions:
    - name: create-servicenow-ticket
      provider:
        type: servicenow
        config: "{{ providers.servicenow }}"
        with:
          table_name: INCIDENT
          payload:
            short_description: "Alert: {{ alert.name }}"
            description: "{{ alert.description }}"
      enrich_alert:
        - key: ticket_id
          value: "{{ results.sys_id }}"
        - key: ticket_url
          value: "{{ results.link }}"

```

## 增强故障排查


### 没有告警/事件时的增强
如果触发器中没有告警/事件（例如间隔触发器或在工作流页面中手动调用），增强规则将没有可应用的告警/事件。增强过程通常需要一个告警/事件存在才能应用指定的增强内容。没有告警/事件时，增强规则将无法按预期执行。一种变通方法是使用 foreach 指令并传入一个包含 "fingerprint" 变量的对象。
