Keep 工作流引擎旨在通过集成触发器、步骤、动作和条件来简化和自动化运维任务。本文档概述了用于定义和执行工作流的核心概念。


### 总体结构


每个工作流由以下部分组成：
1. **元数据（metadata）** - id, description
2. **触发器（triggers）** - 此工作流何时运行？
3. **步骤/动作（steps/actions）** - 此工作流应该做什么？

工作流的总体结构如下：

```yaml
workflow:
  id: aks-example
  description: aks-example
  triggers:
    # list of triggers
    - type: manual
  steps:
    # list of steps
    - name: some-step
      provider:
        type: some-provider-type
        config: "{{ providers.provider_id }}"
        with:
          # provider configuration
    - ...
  actions:
    - name: some-action
      provider:
        type: some-provider-type
        with:
          # provider configuration
    - ...
```

### 触发器

定义工作流的启动方式，例如手动触发、按计划执行或响应告警，并可附带针对特定条件的可选过滤器。

[查看语法](/workflows/syntax/triggers)

### 步骤和动作

表示顺序执行的操作，例如查询数据或运行脚本，使用可配置的提供者。

[查看语法](/workflows/syntax/steps-and-actions)

### 条件

允许基于阈值、断言或前一步骤结果在动作中进行决策。

[查看语法](/workflows/syntax/conditions)

### 函数

内置辅助函数，如 datetime_compare 或 is_business_hours，可简化复杂操作。

[查看语法](/workflows/syntax/functions)

### 上下文

支持在动作或条件中访问和重用前序步骤的输出结果。

[查看语法](/workflows/syntax/context)

### 提供者

通过标准配置接口集成到工作流中的外部系统或服务（例如 Slack、Datadog、ServiceNow）。

[查看语法](/workflows/syntax/providers)

### Foreach 循环

遍历步骤返回的结果列表，对每个元素执行重复操作。

[查看语法](/workflows/syntax/foreach)

### 告警增强

使用步骤或动作中的增强机制，为告警添加上下文信息，如客户详情或工单元数据。

[查看语法](/workflows/syntax/enrichment)
