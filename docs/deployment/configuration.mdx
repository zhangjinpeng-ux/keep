## 背景

Keep 通过环境变量进行高度可配置。这使您无需修改代码即可自定义后端和前端组件的各个方面。环境变量可以在您的部署环境中设置，例如在 Kubernetes 配置、Docker Compose 文件或直接在您的主机系统上设置。

## 后端环境变量

### 服务网格（内部告警接入）

<Info>
  这些设置允许同一 Kubernetes 集群内的受信任服务向 Keep POST 告警，而无需 Keep API 密钥。这适用于服务间通信，通过网络级认证（例如 Istio mTLS 配合 AuthorizationPolicy）确保只有授权的调用方才能访问 Keep 的告警接入端点。
</Info>

|                   环境变量                    |                                   用途                                    | 是否必需 | 默认值 |     有效选项     |
| :------------------------------------------: | :--------------------------------------------------------------------------: | :------: | :-----------: | :-----------------: |
| **KEEP_ALLOW_MESH_ALERT_INGESTION**          | 允许未经认证的 POST 请求访问 `/alerts/event*` 端点           |    否    |    "false"    | "true" 或 "false"   |

当 `KEEP_ALLOW_MESH_ALERT_INGESTION` 设置为 `"true"` 时，对 `/alerts/event*` 的请求如果没有携带 API 密钥或 bearer 令牌，将被接受并认证为具有 `webhook` 角色的内部服务。

调用服务可以选择设置 `X-Service-Name` HTTP 头以在 Keep 的日志和审计追踪中标识自己：

```bash
curl -X POST http://keep-backend:8080/alerts/event \
  -H "Content-Type: application/json" \
  -H "X-Service-Name: my-service" \
  -d '[{"id":"alert-1","name":"Example Alert","severity":"info","status":"firing","source":["my-service"]}]'
```

认证实体将具有：
- **email**: `service:<X-Service-Name header value>`（如果未设置该头部，则默认为 `service:unknown`）
- **role**: `webhook`（授予 `write:alert` 和 `write:incident` 作用域）

<Warning>
  此功能绕过了告警接入端点的 API 密钥认证。您**必须**将其与网络级访问控制（如 Istio AuthorizationPolicy）配合使用，以限制哪些服务可以访问这些端点。如果没有网络级强制措施，任何可以访问 Keep 后端的客户端都可以 POST 告警。
</Warning>

### 速率限制

<Info>
  速率限制配置控制在指定时间段内可以对 Keep 的 API 端点发出多少请求。这有助于防止滥用并确保系统稳定性。
</Info>

|          环境变量           |                用途                | 是否必需 | 默认值 |                                     有效选项                                     |
| :------------------------: | :-----------------------------------: | :------: | :-----------: | :-----------------------------------------------------------------------------------: |
|    **KEEP_USE_LIMITER**    |   启用或禁用速率限制   |    否    |    "false"    |                                   "true" 或 "false"                                   |
| **KEEP_LIMIT_CONCURRENCY** | 设置 API 端点的速率限制 |    否    | "100/minute"  | 格式: "{number}/{interval}"，其中 interval 可以是 "second", "minute", "hour", "day" |

<Note>
目前，速率限制应用于以下端点：
- POST `/alerts/event` - 通用事件接入端点
- POST `/alerts/{provider_type}` - 特定提供商的事件接入端点

当 `KEEP_USE_LIMITER` 启用时，这些端点根据 `KEEP_LIMIT_CONCURRENCY` 设置进行速率限制。

</Note>
