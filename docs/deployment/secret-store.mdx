## 密钥管理器工厂

`SecretManagerFactory` 是一个实用类，用于创建不同类型的密钥管理器实例。它利用工厂设计模式，根据所需的密钥管理器类型抽象创建逻辑。该工厂支持创建 File、GCP、Kubernetes 和 Vault 密钥管理器的实例。

`SECRET_MANAGER_TYPE` 环境变量在 SecretManagerFactory 中起着关键作用，用于确定在没有为方法调用提供特定类型时默认实例化的密钥管理器类型。

**功能**：

**默认密钥管理器**：如果设置了 `SECRET_MANAGER_TYPE` 环境变量，其值决定工厂将创建的默认密钥管理器类型。
此变量的值应对应于 SecretManagerTypes 枚举中定义的类型之一（`FILE`、`AWS`、`GCP`、`K8S`、`VAULT`、`DB`）。

**配置示例**：

在环境中设置 `SECRET_MANAGER_TYPE=GCP` 将使工厂默认创建 GcpSecretManager 的实例。
如果 `SECRET_MANAGER_TYPE` 未设置或设置为 `FILE`，工厂默认创建 FileSecretManager 的实例。
此环境变量提供了灵活性和配置的便捷性，允许在不同环境或场景中使用不同的密钥管理器，而无需更改代码。

## 文件密钥管理器

`FileSecretManager` 是 BaseSecretManager 的具体实现，用于管理存储在文件系统中的密钥。它使用指定的目录（默认为 ./）来读取、写入和删除密钥文件。

配置：

设置环境变量 `SECRET_MANAGER_DIRECTORY` 以指定密钥存储的目录。如果未设置，默认为当前目录 (./)。

用法：

- 密钥以文件形式存储在指定目录中。
- 读取密钥涉及从文件中获取内容。
- 写入密钥会创建或更新具有给定内容的文件。
- 删除密钥会删除相应的文件。

## AWS 密钥管理器

`AwsSecretManager` 与 Amazon Web Services 的 Secrets Manager 服务集成，用于安全密钥管理。它为在 AWS 环境中存储和管理密钥提供了强大的解决方案。

配置：

所需的环境变量：

- `AWS_REGION`：密钥存储的 AWS 区域
- 对于本地开发：
  - `AWS_ACCESS_KEY_ID`：您的 AWS 访问密钥
  - `AWS_SECRET_ACCESS_KEY`：您的 AWS 秘密访问密钥
    可选：
- `AWS_KMS_KEY_ID`：用于加密密钥的 KMS 密钥 ID
- `AWS_SECRET_MANAGER_TAGS`：要添加到 AWS Secrets Manager 中密钥的逗号分隔标签列表，例如 `key=value,key2=value2`
- `AWS_SECRET_ROTATION_ENABLED`：设置为 `true` 以启用密钥自动轮换（默认：`false`）
- `AWS_SECRET_ROTATION_DAYS`：自动轮换之间的天数（默认：`30`）
- `AWS_SECRET_ROTATION_LAMBDA_ARN`：用于密钥轮换的 Lambda 函数的 ARN，如果启用了轮换则必需

用法：

- 使用 AWS Secrets Manager 服务管理密钥
- 支持创建、更新、读取和删除密钥
- 在创建新密钥时可以自动配置密钥轮换策略

### AWS 密钥轮换

密钥轮换是一种安全最佳实践，可定期自动更新密钥。启用后，Keep 将使用轮换计划配置新创建的密钥。

要使用密钥轮换：

1. 创建一个用于轮换密钥的 Lambda 函数（AWS 提供了常见轮换场景的蓝图）
2. 在您的环境中设置 `AWS_SECRET_ROTATION_ENABLED=true`
3. 设置 `AWS_SECRET_ROTATION_LAMBDA_ARN` 为您的轮换 Lambda 函数的 ARN
4. 可选地设置 `AWS_SECRET_ROTATION_DAYS` 来自定义轮换间隔

Lambda ARN 格式示例：`arn:aws:lambda:region:account-id:function:function-name`

注意：不同的密钥类型（数据库凭证、API 密钥等）需要不同的轮换逻辑。请确保您的 Lambda 函数适合您存储的密钥类型。

## Kubernetes 密钥管理器

### 概述

`KubernetesSecretManager` 与 Kubernetes 的原生密钥系统对接。

它在指定的 Kubernetes 命名空间内管理密钥，并设计为在 Kubernetes 集群内运行。

### 配置

- `SECRET_MANAGER_TYPE=k8s`
- `K8S_NAMESPACE=keep` - 指定 Kubernetes 命名空间的环境变量。如果未设置，默认为 `.metadata.namespace`。假设在集群内运行时 Kubernetes 配置（如服务账号令牌）已正确设置。
- `K8S_VERIFY_SSL_CERT=true` - 指定是否验证 Kubernetes API 的 SSL 证书的环境变量。默认为 `true`。

用法：

- 密钥以 Kubernetes Secret 对象的形式存储。
- 提供创建、检索和删除 Kubernetes 密钥的功能。
- 处理 Kubernetes 所需的 base64 编码和解码。

### 从密钥中获取环境变量

Kubernetes 密钥管理器集成允许 Keep 从 Kubernetes Secrets 中获取环境变量。

对于敏感的环境变量，如 `DATABASE_CONNECTION_STRING`，建议作为密钥存储：

#### 创建数据库连接密钥

```bash
# 创建不带换行符的 base64 编码字符串
CONNECTION_STRING_B64=$(echo -n "mysql+pymysql://user:password@host:3306/dbname" | base64)

# 创建 Kubernetes 密钥
kubectl create secret generic keep-db-secret \
  --namespace=keep \
  --from-literal=connection_string=$(echo -n "mysql+pymysql://user:password@host:3306/dbname" | base64)

# 或使用 YAML 文件：
cat <<EOF | kubectl apply -f -
apiVersion: v1
kind: Secret
metadata:
  name: keep-db-secret
  namespace: keep
type: Opaque
data:
  connection_string: $(echo -n "mysql+pymysql://user:password@host:3306/dbname" | base64)
EOF
```

#### 更新 Helm Values.yaml

创建密钥后，更新 `values.yaml` 以便 helm chart 将密钥注入为环境变量：

```bash
backend:
  enabled: true
  waitForDatabase: true
  databaseConnectionStringFromSecret:
    enabled: true  # 启用使用密钥进行数据库连接
    secretName: "keep-db-secret"  # 我们创建的密钥名称
    secretKey: "connection_string"  # 包含连接字符串的密钥中的键
```

#### 使用 Helm 应用

```bash
# 如果是首次安装
helm install keep keephq/keep \
  -f values.yaml \
  --namespace keep

# 如果更新现有安装
helm upgrade keep keephq/keep \
  -f values.yaml \
  --namespace keep
```

#### 验证安装

检查密钥是否正确创建：

```bash
kubectl get secret keep-db-secret -n keep
```

验证密钥内容是否正确：

```bash
kubectl get secret keep-db-secret -n keep -o jsonpath='{.data.connection_string}' | base64 -d
```

验证使用密钥的 Pod：

```bash
kubectl get pod -n keep -l app.kubernetes.io/component=backend -o yaml | grep DATABASE_CONNECTION_STRING -A 5
```

## GCP 密钥管理器

`GcpSecretManager` 利用 Google Cloud 的 Secret Manager 服务进行密钥管理。它需要设置 Google Cloud 凭证和项目 ID。

配置：

确保环境变量 `GOOGLE_CLOUD_PROJECT` 已设置为您的 Google Cloud 项目 ID。

用法：

- 使用 Google Cloud 的 Secret Manager 管理密钥。
- 支持在云中创建、访问和删除密钥的操作。
- 与 OpenTelemetry 集成以追踪密钥管理操作。

## Hashicorp Vault 密钥管理器

`VaultSecretManager` 专为 Hashicorp Vault 量身定制，Vault 是管理敏感数据的工具。它支持基于令牌的认证以及基于 Kubernetes 的 Vault 认证。

配置：

- 设置 `HASHICORP_VAULT_ADDR` 为 Vault 服务器地址。默认为 http://localhost:8200。
- 使用 `HASHICORP_VAULT_TOKEN` 进行基于令牌的认证。
- 将 `HASHICORP_VAULT_USE_K8S` 设置为 True 并提供 `HASHICORP_VAULT_K8S_ROLE` 用于基于 Kubernetes 的认证。

用法：

- 在 Hashicorp Vault 服务器中管理密钥。
- 提供从 Vault 写入、读取和删除密钥的方法。
- 支持不同的 Vault 认证方法，包括静态令牌和 Kubernetes 服务账号令牌。

## DB 密钥管理器

`DbSecretManager` 是 BaseSecretManager 的具体实现，用于管理存储在数据库中的密钥。它使用 `secret` 表来读取、写入和删除密钥。

配置：

确保 `secret` 表存在。

用法：

- 密钥存储在 `secret` 表中。
