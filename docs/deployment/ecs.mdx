## 步骤 1: 登录 AWS 控制台
- 打开 Web 浏览器并导航到 AWS Management Console。
- 使用您的 AWS 账户凭证登录。

## 步骤 2: 导航到 ECS
- 点击左上角的 "Services" 下拉菜单。
- 从服务列表中选择 "ECS"。

## 步骤 3: 创建 3 个任务定义
- 在 ECS 仪表板中，导航到左侧边栏的 "Task Definitions" 部分。
    <img src="/images/ecs-task-def-create.png" alt="任务定义" width="200" height="200" />
- 点击 "Create new Task Definition"。
    ![创建新的任务定义](/images/ecs-task-def-create-new.png)

    ### 任务定义 1 (前端 - KeepUI):

    - 任务定义系列: keep-frontend
        ![任务定义系列](/images/ecs-task-def-frontend1.png)
    - 按照以下配置您的容器定义：
        - 基础设施需求:
            - 启动类型: AWS Fargate
            - 操作系统、架构、网络模式: Linux/X86_64
            - 任务大小:
                - CPU: 1 vCPU
                - Memory: 2 GB
            - 任务角色和任务执行角色是可选的，例如如果您计划使用密钥管理器，则需要创建一个任务执行角色以允许访问您创建的密钥管理器。
        ![基础设施需求](/images/ecs-task-def-frontend2.png)
        - 容器详情:
            - 名称: keep-frontend
            - 镜像 URI: us-central1-docker.pkg.dev/keephq/keep/keep-api:latest
            - 端口映射:
                - 容器端口: 3000
                - 协议: TCP
            ![容器详情](/images/ecs-task-def-frontend3.png)
            - 环境变量: (可以是静态的，也可以使用参数存储或密钥管理器)
                - DATABASE_CONNECTION_STRING
                - AUTH_TYPE
                - KEEP_JWT_SECRET
                - KEEP_DEFAULT_USERNAME
                - KEEP_DEFAULT_PASSWORD
                - SECRET_MANAGER_TYPE
                - SECRET_MANAGER_DIRECTORY
                - USE_NGROK
                - KEEP_API_URL
                （以下变量如果您不想使用 websocket，则为可选）
                - PUSHER_DISABLED
                （以下变量如果您想使用 websocket，则为可选）
                - PUSHER_APP_ID
                - PUSHER_APP_KEY
                - PUSHER_APP_SECRET
                - PUSHER_HOST
                - PUSHER_PORT
            ![环境变量](/images/ecs-task-def-frontend4.png)
        - 检查并创建您的任务定义。

                ![基础设施需求](/images/ecs-task-def-backend2.png)

            - 环境变量: (可以是静态的，也可以使用参数存储或密钥管理器)
                - DATABASE_CONNECTION_STRING

                - KEEP_JWT_SECRET
                - KEEP_DEFAULT_USERNAME
                - KEEP_DEFAULT_PASSWORD
                - SECRET_MANAGER_TYPE
                - SECRET_MANAGER_DIRECTORY
                - USE_NGROK
                - KEEP_API_URL
                （以下变量如果您不想使用 websocket，则为可选）

                （以下变量如果您想使用 websocket，则为可选）

## 步骤 4: 创建 Keep 服务
- 在 ECS 仪表板中，导航到左侧边栏的 "Clusters" 部分。
- 选择您要部署服务的集群。
- 点击 "Services" 旁边的 "Create" 按钮。
- 配置您的服务设置。
- 检查并创建您的服务。

## 步骤 5: 监控您的服务
- 创建服务后，在 ECS 仪表板中监控其状态。
- 您可以查看任务状态、服务事件和其他指标，以确保您的服务正常运行。
