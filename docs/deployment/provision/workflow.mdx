Keep 中的工作流预置允许您动态设置和管理工作流。此功能使您能够在 Keep 部署中配置各种自动化流程和任务。

### 配置工作流

要预置工作流，请按照以下步骤操作：

1. 将 `KEEP_WORKFLOWS_DIRECTORY` 环境变量设置为工作流配置目录的路径。
2. 在指定目录中创建工作流配置文件。

目录结构示例：
```
/path/to/workflows/
├── workflow1.yaml
├── workflow2.yaml
└── workflow3.yaml
```
### 更新已预置的工作流

每次重启时，Keep 读取 `KEEP_WORKFLOWS_DIRECTORY` 环境变量并确定哪些工作流需要添加、移除或更新。

此过程允许灵活管理工作流，无需人工干预。通过简单地更新 `KEEP_WORKFLOWS_DIRECTORY` 中的工作流文件并重启应用程序，您可以高效地添加新工作流、移除现有工作流或修改其配置。

高层预置机制：
1. Keep 读取 `KEEP_WORKFLOWS_DIRECTORY` 的值。
2. Keep 列出 `KEEP_WORKFLOWS_DIRECTORY` 目录下的所有工作流文件。
3. Keep 将当前工作流文件与之前预置的工作流进行比较：
   - 新的工作流文件被预置。
   - 缺失的工作流文件被取消预置。
   - 更新的工作流文件使用新配置重新预置。
4. Keep 更新其内部状态以反映当前预置的工作流集合。
