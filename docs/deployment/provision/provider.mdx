title: "Providers Provisioning"

<Tip>对于任何与提供商预置相关的问题，请加入我们的 [Slack](https://slack.keephq.dev) 社区。</Tip>

Keep 中的提供商预置允许您动态设置和管理数据提供商。此功能使您能够配置 Keep 可以交互的各种数据源，例如监控系统、数据库或其他服务。

### 配置提供商

要预置提供商及其去重规则，我们可以通过环境变量进行配置。这可以通过两种方式进行：
1. 使用 `KEEP_PROVIDERS` 环境变量，其中包含一个 JSON 字符串或指向包含提供商配置的 JSON 文件的路径。
2. 使用 `KEEP_PROVIDERS_DIRECTORY` 环境变量，其中包含指向包含提供商配置的目录的路径（通过 YAML 文件配置）。这是推荐的方式。

<Note>
Keep 不允许同时使用 `KEEP_PROVIDERS` 和 `KEEP_PROVIDERS_DIRECTORY` 环境变量。
</Note>

<Note>
Keep 可以自动为支持 webhook 的提供商安装 webhook。此行为取决于配置和使用的预置方法。
</Note>

<Tip>请注意：去重规则对于提供商分发不是强制性的。</Tip>

### 使用 KEEP_PROVIDERS 预置提供商

提供商预置 JSON 示例：
```json
{
  "keepVictoriaMetrics": {
    "type": "victoriametrics",
    "authentication": {
      "VMAlertHost": "http://localhost",
      "VMAlertPort": 1234
    },
    "install_webhook": true,
    "deduplication_rules": {
      "deduplication rule name example 1": {
        "description": "deduplication rule name example 1",
        "fingerprint_fields": ["fingerprint", "source", "service"],
        "full_deduplication": true,
        "ignore_fields": ["name", "lastReceived"]
      },
      "deduplication rule name example 2": {
        "description": "deduplication rule name example 2",
        "fingerprint_fields": ["fingerprint", "source", "service"],
        "full_deduplication": false,
      }
    }
  },
  "keepClickhouse1": {
    "type": "clickhouse",
    "authentication": {
      "host": "http://localhost",
      "port": 1234,
      "username": "keep",
      "password": "keep",
      "database": "keep-db"
    }
  }
}
```

使用此 `KEEP_PROVIDERS` 值启动 Keep：
```json
# ENV
KEEP_PROVIDERS={"keepVictoriaMetrics":{"type":"victoriametrics","authentication":{"VMAlertHost":"http://localhost","VMAlertPort": 1234},"install_webhook":true},"keepClickhouse1":{"type":"clickhouse","authentication":{"host":"http://localhost","port":"4321","username":"keep","password":"1234","database":"keepdb"}}}
```

默认情况下，当使用 `KEEP_PROVIDERS` 预置时，除非 `install_webhook` 标志设置为 `false`，否则 webhook 会自动为支持它们的提供商安装。

### 使用 KEEP_PROVIDERS_DIRECTORY 预置提供商

指定包含提供商配置的目录路径：

```bash
# ENV
KEEP_PROVIDERS_DIRECTORY=/path/to/providers
```

该目录应包含提供商配置的 YAML 文件。

提供商配置 YAML 文件示例：

```yaml
name: keepVictoriaMetrics
type: victoriametrics
authentication:
  VMAlertHost: http://localhost
  VMAlertPort: 1234
install_webhook: false
deduplication_rules:
  deduplication_rule_name_example_1:
    description: deduplication rule name example 1
    fingerprint_fields:
      - fingerprint
      - source
      - service
    full_deduplication: true
    ignore_fields:
      - name
      - lastReceived
```

`install_webhook` 字段控制 Keep 是否自动为该提供商设置 webhook。默认情况下，当使用 `KEEP_PROVIDERS_DIRECTORY` 预置时，webhook 安装是禁用的，除非显式设置为 `true`。

### 支持的提供商

Keep 支持多种提供商类型。每种提供商类型都有其特定的配置要求。
要查看支持的提供商的完整列表及其详细配置选项，请参阅我们全面的提供商文档。


### 更新已预置的提供商

#### 使用 KEEP_PROVIDERS

提供商配置可以通过更改 `KEEP_PROVIDERS` 环境变量来动态更新。

每次重启时，Keep 读取此环境变量并确定需要添加或移除哪些提供商。

此过程允许灵活管理数据源，无需人工干预。通过简单地更新 `KEEP_PROVIDERS` 变量并重启应用程序，您可以高效地添加新的提供商、移除现有的提供商或修改其配置。

高层预置机制：
1. Keep 读取 `KEEP_PROVIDERS` 的值。
2. Keep 检查是否有任何已预置的提供商不再存在于 `KEEP_PROVIDERS` 值中，并将其删除。
3. Keep 从 `KEEP_PROVIDERS` 值安装所有提供商。

#### 使用 KEEP_PROVIDERS_DIRECTORY

提供商配置可以通过更改 `KEEP_PROVIDERS_DIRECTORY` 目录中的 YAML 文件来动态更新。

每次重启时，Keep 读取 `KEEP_PROVIDERS_DIRECTORY` 目录中的 YAML 文件并确定需要添加或移除哪些提供商。

高层预置机制：
1. Keep 读取 `KEEP_PROVIDERS_DIRECTORY` 目录中的 YAML 文件。
2. Keep 检查是否有任何已预置的提供商不再存在于 YAML 文件中，并将其删除。
3. Keep 从 YAML 文件安装所有提供商。
