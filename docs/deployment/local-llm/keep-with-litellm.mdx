## 概述

本指南将帮助您使用 LiteLLM 设置 Keep，LiteLLM 是一个支持超过 100 个 LLM 提供商的多功能工具。LiteLLM 充当遵循 OpenAI 标准的代理，允许与 Keep 无缝集成。通过遵循本指南，您可以轻松配置 Keep 使用 LiteLLM 与各种 LLM 提供商配合工作。

### 动机

将 LiteLLM 与 Keep 结合使用，使组织能够在本地部署和气隙环境中运行本地模型。此设置特别有利于利用 AIOps 功能，同时确保敏感数据不会离开本地环境。通过使用 LiteLLM 作为代理，您可以与 Keep 无缝集成，并访问各种 LLM 提供商，而不会损害数据安全。此方法非常适合重视数据隐私并需要遵守严格监管要求的组织。

## 前提条件

### 在本地运行 LiteLLM

1. 确保您的系统上已安装 Python 和 pip。
2. 运行以下命令安装 LiteLLM：

```bash
pip install litellm
```

3. 使用您所需的模型启动 LiteLLM。例如，使用 HuggingFace 模型：

```bash
litellm --model huggingface/bigcode/starcoder
```

这将在 `http://0.0.0.0:4000` 上启动代理服务器。

### 使用 Docker 运行 LiteLLM

要使用 Docker 运行 LiteLLM，可以使用以下命令：

```bash
docker run -p 4000:4000 litellm/litellm --model huggingface/bigcode/starcoder
```

此命令将在 Docker 容器中启动 LiteLLM 代理，暴露在端口 4000 上。

## 配置

|           Env var           |                   Purpose                   | Required | Default Value |               Valid options               |
| :-------------------------: | :-----------------------------------------: | :------: | :-----------: | :---------------------------------------: |
| **OPEN_AI_ORGANIZATION_ID** | OpenAI/LiteLLM 服务的组织 ID |   是    |     无     |       有效的组织 ID 字符串        |
|     **OPEN_AI_API_KEY**     |     OpenAI/LiteLLM 服务的 API 密钥     |   是    |     无     |           有效的 API 密钥字符串            |
|     **OPENAI_BASE_URL**     |       LiteLLM 代理的基础 URL        |   是    |     无     | 有效的 URL (e.g., "http://localhost:4000") |

<Note>
  这些环境变量应在 Keep 的**前端**和**后端**都设置。
</Note>
