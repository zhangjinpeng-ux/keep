推荐的 Keep 在 Kubernetes 上的安装方式是通过 Helm Chart。<br></br>

</Tip>

# 前提条件

## Helm CLI
有关安装 Helm 的说明，请参阅 [Helm 文档](https://helm.sh/docs/intro/install/)。

## Ingress 控制器（可选）
<Info>
以下情况可以跳过此步骤：
1. 您已经安装了 **ingress-nginx**。
2. 您不需要将 Keep 暴露到互联网/网络。
</Info>

### 概述
Ingress 控制器对于管理 Kubernetes 集群中服务的外部访问至关重要。它充当智能路由器和负载均衡器，允许您通过单一入口点暴露多个服务，同时处理 SSL 终止和路由规则。



**Keep 与** [ingress-nginx](https://github.com/kubernetes/ingress-nginx) **和** [HAProxy Ingress](https://haproxy-ingress.github.io/) **控制器配合使用效果最佳，但您也可以为其他 Ingress 控制器自定义 Helm chart。**


### Nginx Ingress 控制器

#### 检查 ingress-nginx 是否已安装
检查是否已安装 ingress-nginx：
```bash
# 默认情况下，ingress-nginx 将安装在 ingress-nginx 命名空间下
kubectl -n ingress-nginx get pods
NAME                                       READY   STATUS    RESTARTS   AGE
ingress-nginx-controller-d49697d5f-hjhbj   1/1     Running   0          4h19m

# 或检查 ingress class
kubectl get ingressclass
NAME    CONTROLLER             PARAMETERS   AGE
nginx   k8s.io/ingress-nginx   <none>       4h19m

```

#### 安装 ingress-nginx
<Info>
要了解更多安装选项，请参阅 [ingress-nginx 安装文档](https://kubernetes.github.io/ingress-nginx/deploy/)。
</Info>
<Tip>
自 ingress-nginx 4.12 起，您需要添加
```
--set controller.config.annotations-risk-level=Critical
```
参见 https://github.com/kubernetes/ingress-nginx/issues/12618#issuecomment-2566084202
</Tip>
```bash
# 最简单的安装方式
# 我们设置 snippet-annotations 为 true 以允许重写
#   参见 https://kubernetes.github.io/ingress-nginx/user-guide/nginx-configuration/configmap/#allow-snippet-annotations
helm upgrade --install ingress-nginx ingress-nginx \
  --repo https://kubernetes.github.io/ingress-nginx \
  --set controller.config.allow-snippet-annotations=true \
  --set controller.config.annotations-risk-level=Critical \
  --namespace ingress-nginx --create-namespace
```

验证安装：
```bash
kubectl get ingressclass
NAME    CONTROLLER             PARAMETERS   AGE
nginx   k8s.io/ingress-nginx   <none>       4h19m
```

验证 snippet 注解是否启用：
```bash
kubectl get configmap -n ingress-nginx ingress-nginx-controller -o yaml | grep allow-snippet-annotations
allow-snippet-annotations: "true"
```

### HAProxy Ingress 控制器

#### 安装 ingress-haproxy
<Info>
要了解更多安装选项，请参阅 [haproxy-ingress 安装文档](https://haproxy-ingress.github.io/docs/getting-started/)。
</Info>

```bash
# 最简单的安装方式
helm upgrade --install haproxy-ingress haproxy-ingress \
  --repo https://haproxy-ingress.github.io/charts \
  --namespace ingress-haproxy --create-namespace
```

验证安装：
```bash
kubectl get ingressclass
NAME      CONTROLLER                      PARAMETERS   AGE
haproxy   haproxy-ingress.github.io/controller        <none>        4h19m
```

验证控制器是否正在运行：
```bash
kubectl get pods -n ingress-haproxy -l app.kubernetes.io/instance=haproxy-ingress
NAME                                READY   STATUS    RESTARTS   AGE
haproxy-ingress-controller-x4n2z   1/1     Running   0          4h19m
```

## 安装

### 使用 Ingress-NGINX（推荐）

```bash
# 添加 Helm 仓库
helm repo add keephq https://keephq.github.io/helm-charts

# 安装 Keep 并启用 ingress
helm install keep keephq/keep -n keep --create-namespace
```

### 使用 Ingress-HAProxy（推荐）

```bash
# 添加 Helm 仓库
helm repo add keephq https://keephq.github.io/helm-charts

# 安装 Keep 并启用 ingress
helm install keep keephq/keep -n keep --create-namespace --set global.ingress.className=haproxy
```

### 不使用 Ingress（不推荐）

```bash
# 添加 Helm 仓库
helm repo add keephq https://keephq.github.io/helm-charts

# 安装 Keep 但不启用 ingress。
# 您将无法从网络访问 Keep。
helm install keep keephq/keep -n keep --create-namespace \
  --set global.ingress.enabled=false
```

## 访问 Keep

### Ingress
如果您使用 ingress 安装了 Keep，您应该能够访问 Keep。

```bash
kubectl -n keep get ingress
NAME           CLASS   HOSTS   ADDRESS        PORTS   AGE
keep-ingress   nginx   *       X.X.X.X        80      4h16m
```

Keep 可在 http://X.X.X.X 访问 :)

### 不使用 Ingress（端口转发）

使用以下命令在本地访问 Keep（不使用 ingress）：
```bash
# 转发 UI
kubectl port-forward svc/keep-frontend 3000:3000 -n keep &

# 转发后端
kubectl port-forward svc/keep-backend 8080:8080 -n keep &

# 转发 WebSocket 服务器（可选）
kubectl port-forward svc/keep-websocket 6001:6001 -n keep &
```

Keep 可在 http://localhost:3000 访问 :)

## 配置 HTTPS

### 前提条件
1. 域名：例如 - keep.yourcompany.com
2. TLS 证书：私钥 (tls.key) 和证书 (tls.crt)

### 创建 TLS Secret

假设：
- `tls.crt` 包含证书。
- `tls.key` 包含私钥。

```bash
# 使用 kubectl 创建 secret
kubectl create secret tls keep-tls --cert=./tls.crt --key=./tls.key -n keep
```

### 更新 Helm Values 以配置 TLS
```bash
helm upgrade -n keep keep keephq/keep \
  --set "global.ingress.hosts[0].host=keep.example.com" \
  --set "global.ingress.tls[0].hosts[0]=keep.example.com" \
  --set "global.ingress.tls[0].secretName=keep-tls"
```



或者，更新您的 `values.yaml`：
```bash
...
global:
  ingress:
    hosts:
      - host: keep.example.com
    tls:
      - hosts:
          - keep.example.com
        secretName: keep-tls
...
```


## 卸载
要移除 Keep 并清理：
```bash
helm uninstall keep -n keep
kubectl delete namespace keep
```
