### 使用 docker-compose 最新镜像启动 Keep
启动 Keep 最简单的方式是使用 docker-compose：
```shell
curl https://raw.githubusercontent.com/keephq/keep/main/start.sh | sh
```

```bash start.sh
#!/bin/bash
# Keep install script for docker compose
set -e

echo "Creating state directory."
mkdir -p state
test -e state || echo "Unable to create folder"
echo "Changing directory ownership to non-privileged user."
chown -R 999:999 state || echo "Unable to change directory ownership, changing permissions instead." && chmod -R 0777 state
which curl &> /dev/null || echo "curl not installed"
curl https://raw.githubusercontent.com/keephq/keep/main/docker-compose.yml --output docker-compose.yml
curl https://raw.githubusercontent.com/keephq/keep/main/docker-compose.common.yml --output docker-compose.common.yml

docker compose up -d
```

docker-compose.yml 包含 3 个服务：
- [keep-backend](https://console.cloud.google.com/artifacts/docker/keephq/us-central1/keep/keep-api?project=keephq) - 一个作为 API 服务器的 FastAPI 服务。
- [keep-frontend](https://console.cloud.google.com/artifacts/docker/keephq/us-central1/keep/keep-ui?project=keephq) - 一个作为 Keep UI 界面的 Next.js 应用。
- [keep-websocket-server](https://docs.soketi.app/getting-started/installation/docker) - Soketi（兼容 pusher 的 WebSocket 服务器）用于实时告警。

### 全新重新安装 Keep

`注意：` 此操作将从头开始刷新，Keep 的数据和设置将被清除。此主机上的其他容器也会被清除。因此在使用以下步骤时请仔细考虑。

适用于您需要测试许多不同选项，或只想使用 docker compose 从头重新安装 Keep 而无需花费太多时间的情况，即无需重复安装 docker、下载安装程序等步骤。

请按照以下步骤操作

#### 步骤1：停止、清除容器、网络、卷、镜像。
在包含已下载的 docker compose 文件的目录中，比如 `/root/`

```
docker-compose down

docker-compose down --rmi all

docker-compose down -v

docker system prune -a --volumes
```

#### 步骤2：清除 state 文件夹中的配置数据库、配置文件。

```
rm -rf state/*

```

#### 步骤 3：重新运行

```
docker compose up -d
```
