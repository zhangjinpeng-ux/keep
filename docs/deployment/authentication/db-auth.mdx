对于需要用户管理和认证的应用程序，Keep 支持使用用户名和密码的基本认证。

### 何时使用

- **自托管部署：** 当您为个人使用或在组织内部署 Keep 时。
- **增强安全性：** 为您的 Keep 实例提供简单而有效的安全层。

### 设置说明

要使用 DB 认证启动 Keep，请设置以下环境变量：

| 环境变量 | 描述 | 是否必需 | 前端/后端 | 默认值 |
|--------------------|:-----------:|:--------:|:----------------:|:-------------:|
| AUTH_TYPE | 设置为 'DB' 以启用数据库认证 | 是 | 两者 | - |
| KEEP_JWT_SECRET | JWT 令牌生成的密钥 | 是 | 后端 | - |
| KEEP_DEFAULT_USERNAME | 默认管理员用户名 | 否 | 后端 | keep |
| KEEP_DEFAULT_PASSWORD | 默认管理员密码 | 否 | 后端 | keep |
| KEEP_FORCE_RESET_DEFAULT_PASSWORD | 覆盖当前管理员密码 | 否 | 后端 | false |

### 配置示例

使用 `docker-compose-with-auth.yml` 进行简单设置，其中包含启用基本认证所需的环境变量。
