Keep 通过 Auth0 支持多租户环境，使独立的租户能够在同一 Keep 平台中独立运行。

  <img src="/images/auth0auth.png" width="500"/>

### 何时使用

- **已在使用 Auth0：** 如果您的组织已在用 Auth0，您可以将其作为 Keep 的认证提供商。
- **SSO/SAML：** Auth0 支持各种 Single Sign-On (SSO) 和 SAML 协议，允许您将 Keep 与现有的身份管理系统集成。

### 设置说明

要使用 Auth0 认证启动 Keep，请设置以下环境变量：

#### 前端环境变量

| 环境变量 | 描述 | 是否必需 | 默认值 |
|--------------------|-----------|:--------:|:-------------:|
| AUTH_TYPE | 设置为 'AUTH0' 以启用 Auth0 认证 | 是 | - |
| AUTH0_DOMAIN | 您的 Auth0 域名 | 是 | - |
| AUTH0_CLIENT_ID | 您的 Auth0 客户端 ID | 是 | - |
| AUTH0_CLIENT_SECRET | 您的 Auth0 客户端密钥 | 是 | - |
| AUTH0_ISSUER | 您的 Auth0 API 签发者 | 是 | - |

#### 后端环境变量

| 环境变量 | 描述 | 是否必需 | 默认值 |
|--------------------|-----------|:--------:|:-------------:|
| AUTH_TYPE | 设置为 'AUTH0' 以启用 Auth0 认证 | 是 | - |
| AUTH0_MANAGEMENT_DOMAIN | 您的 Auth0 管理域名 | 是 | - |
| AUTH0_CLIENT_ID | 您的 Auth0 客户端 ID | 是 | - |
| AUTH0_CLIENT_SECRET | 您的 Auth0 客户端密钥 | 是 | - |
| AUTH0_AUDIENCE | 您的 Auth0 API 受众 | 是 | - |

### 配置示例

使用 `docker-compose-with-auth0.yml` 进行简单设置，其中包含启用 Auth0 认证所需的环境变量。
