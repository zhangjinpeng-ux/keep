---
title: "No Authentication"
---
<Warning>在生产环境中使用此配置是不安全的，强烈不建议。</Warning>


部署不带认证的 Keep 是最快速的启动方式，非常适合本地开发或安全性不是问题的内部工具。
## 设置说明
无论您使用 docker-compose、Kubernetes、OpenShift 还是任何其他部署方法，添加以下环境变量：
```
# Frontend
AUTH_TYPE=NOAUTH

# Backend
AUTH_TYPE=NOAUTH
```
## 影响
使用 `AUTH_TYPE=NOAUTH`：
- Keep 不会显示任何登录页面，并允许您无需认证即可使用 API。
- Keep 将使用以 "keep" 作为租户 ID 的 JWT，但不会验证它。
- 在 `x-api-key` 头中提供的任何 API 密钥都将被接受，无需验证。

此配置本质上绕过了所有认证检查，使其不适用于关注安全性的生产环境。
