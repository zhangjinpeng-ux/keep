将认证委托给 Oauth2Proxy。

### 何时使用

- **oauth2-proxy 用户：** 如果您想将认证委托给外部的 Oauth2Proxy 服务，请使用此认证方法。

### 设置说明

要使用 Oauth2Proxy 认证启动 Keep，请设置以下环境变量：

#### 前端环境变量

| 环境变量 | 描述 | 是否必需 | 默认值 |
|--------------------|-----------|:--------:|:-------------:|
| AUTH_TYPE | 设置为 'OAUTH2PROXY' 以启用 OAUTH2PROXY 认证 | 是 | - |
| KEEP_OAUTH2_PROXY_USER_HEADER | 已认证用户邮箱的请求头 | 是 | x-forwarded-email |
| KEEP_OAUTH2_PROXY_ROLE_HEADER | 已认证用户角色的请求头 | 是 | x-forwarded-groups |

#### 后端环境变量

| 环境变量 | 描述 | 是否必需 | 默认值 |
|--------------------|-----------|:--------:|:-------------:|
| AUTH_TYPE | 设置为 'OAUTH2PROXY' 以启用 OAUTH2PROXY 认证 | 是 | - |
| KEEP_OAUTH2_PROXY_USER_HEADER | 已认证用户邮箱的请求头 | 是 | x-forwarded-email |
| KEEP_OAUTH2_PROXY_ROLE_HEADER | 已认证用户角色的请求头 | 是 | x-forwarded-groups |
| KEEP_OAUTH2_PROXY_AUTO_CREATE_USER | 如果用户不存在则自动创建 | 否 | true |
| KEEP_OAUTH2_PROXY_ADMIN_ROLES | 管理员用户的角色名称 | 否 | admin |
| KEEP_OAUTH2_PROXY_NOC_ROLES | NOC（网络运营中心）用户的角色名称 | 否 | noc |
| KEEP_OAUTH2_PROXY_WEBHOOK_ROLES | Webhook 用户的角色名称 | 否 | webhook |
