Keep 通过 Azure Entre ID（前身为 Azure AD）支持企业级认证，使组织能够使用其现有的 Microsoft 身份平台进行安全的访问管理。

## 何时使用

- **Microsoft 环境：** 如果您的组织使用 Microsoft 365 或 Azure 服务，Azure AD 集成可提供无缝认证。
- **企业 SSO：** 利用 Azure AD 的 Single Sign-On 功能实现统一的访问管理。

## 设置说明 (在 Azure AD 上)

### 创建 Azure AD 应用程序

1. 登录 [Azure Portal](https://portal.azure.com)
2. 导航到 **Microsoft Entra ID** > **App registrations** > **New registration**

<Frame>
  <img src="/images/azuread_1.png" width="1000" alt="Azure AD 应用注册"/>
</Frame>

3. 配置应用程序：
   - 名称："Keep"

<Info>请注意，我们使用 "Register an application to integrate with Microsoft Entra ID (App you're developing)"，因为您是自托管 Keep 并且需要直接控制特定实例的认证流程和权限 — 与云端/托管版本不同，后者的 Keep 团队已配置了集中式应用程序注册。</Info>

<Frame>
  <img src="/images/azuread_2.png" width="1000" alt="Azure AD 应用注册"/>
</Frame>

4. 配置应用程序（继续）
- 支持的账户类型："Single tenant"

<Info>
我们建议使用 "Single tenant" 以增强安全性，因为它将访问限制为仅您组织内的用户。虽然可以配置多租户，但这将允许任何 Azure AD 目录中的用户访问您的 Keep 实例，除非您有特定的跨组织需求，否则可能带来安全风险。
</Info>

    - Redirect URI："Web" + 您的重定向 URI

<Info>
我们使用 "Web" 平台而非 "Single Page Application (SPA)"，因为 Keep 的后端使用客户端凭证/密钥处理认证流程，这比 SPA 中使用的隐式流程更安全。这可以防止令牌在浏览器中暴露，并通过服务器端令牌验证和刷新令牌处理提供更强的安全性。
</Info>

<Tip>
对于 localhost，重定向地址应为 http://localhost:3000/api/auth/callback/microsoft-entra-id

对于生产环境，应为类似 http://your_keep_frontend_domain/api/auth/callback/microsoft-entra-id 的地址

</Tip>

<Frame>
  <img src="/images/azuread_3.png" width="1000" alt="Azure AD 应用注册"/>
</Frame>

5. 最后，点击 "register"

### 配置认证
创建应用程序后，让我们配置认证。

1. 进入 "App Registrations" -> "All applications"

<Frame>
  <img src="/images/azuread_4.png" width="1000" alt="Azure AD 认证配置"/>
</Frame>


2. 点击您的应用程序 -> "Add a certificate or secret"

<Frame>
  <img src="/images/azuread_5.png" width="1000" alt="Azure AD 认证配置"/>
</Frame>


3. 点击 "New client secret" 并为其命名

<Frame>
  <img src="/images/azuread_6.png" width="1000" alt="Azure AD 认证配置"/>
</Frame>

4. 保留 "Value"，我们稍后将其用作 `KEEP_AZUREAD_CLIENT_SECRET`

<Frame>
  <img src="/images/azuread_7.png" width="1000" alt="Azure AD 认证配置"/>
</Frame>

### 配置组

Keep 将 Azure AD 组映射到角色，有两个默认组：
1. Admin 组（读写）
2. NOC 组（只读）

要创建这些组，进入 Groups -> All groups 并创建两个组：

<Frame>
  <img src="/images/azuread_16.png" width="1000" alt="Azure AD 认证配置"/>
</Frame>

保留这些组的 Object ID，并将其用作 `KEEP_AZUREAD_ADMIN_GROUP_ID` 和 `KEEP_AZUREAD_NOC_GROUP_ID`。

## 设置说明 (在 Keep 上)

配置 Azure AD 后，您应该拥有以下内容：
1. Azure AD 租户 ID
2. Azure AD 客户端 ID

如何获取：

<Frame>
  <img src="/images/azuread_15.png" width="1000" alt="Azure AD 认证配置"/>
</Frame>

3. Azure AD 客户端密钥 [参见配置认证](#configure-authentication)。
4. Admin 和 NOC（只读）的 Azure AD 组 ID [参见配置组](#configure-groups)。


### 配置

#### 前端

| 环境变量 | 描述 | 是否必需 | 默认值 |
|--------------------|-------------|:---------:|:-------------:|
| AUTH_TYPE | 设置为 'AZUREAD' 以启用 Azure AD 认证 | 是 | - |
| KEEP_AZUREAD_CLIENT_ID | 您的 Azure AD 应用程序（客户端）ID | 是 | - |
| KEEP_AZUREAD_CLIENT_SECRET | 您的客户端密钥 | 是 | - |
| KEEP_AZUREAD_TENANT_ID | 您的 Azure AD 租户 ID | 是 | - |
| NEXTAUTH_URL | 您的 Keep 应用程序 URL | 是 | - |
| NEXTAUTH_SECRET | NextAuth.js 的随机字符串 | 是 | - |

#### 后端

| 环境变量 | 描述 | 是否必需 | 默认值 |
|--------------------|-------------|:---------:|:-------------:|
| AUTH_TYPE | 设置为 'AZUREAD' 以启用 Azure AD 认证 | 是 | - |
| KEEP_AZUREAD_TENANT_ID | 您的 Azure AD 租户 ID | 是 | - |
| KEEP_AZUREAD_CLIENT_ID | 您的 Azure AD 应用程序（客户端）ID | 是 | - |
| KEEP_AZUREAD_ADMIN_GROUP_ID | Keep 管理员（读写）的组 ID | 是 | - |
| KEEP_AZUREAD_NOC_GROUP_ID | Keep NOC（只读）的组 ID | 是 | - |

## 功能与限制

#### 支持的功能
- Single Sign-On (SSO)
- 通过 Azure AD 组实现的基于角色的访问控制
- 多因素认证（在 Azure AD 中配置时）

#### 限制
参见[概述](/deployment/authentication/overview)
