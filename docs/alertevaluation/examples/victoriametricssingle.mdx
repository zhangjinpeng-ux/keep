---
title: "VictoriaMetrics 单告警示例"
---

此示例演示了一个基于指标的简单 CPU 使用率告警：

```yaml
# 此工作流查询 VictoriaMetrics 指标并根据 CPU 使用率创建告警
workflow:
  # 此工作流的唯一标识符
  id: query-victoriametrics
  # UI 中显示的显示名称
  name: victoriametrics-alert-example
  # 此工作流功能的简要描述
  description: Monitors CPU usage metrics from VictoriaMetrics and creates alerts when thresholds are exceeded

  # 定义工作流如何触发
  triggers:
    - type: manual # 可以从 UI 手动触发

  # 按顺序执行的步骤
  steps:
    - name: victoriametrics-step
      provider:
        # 使用 providers.vm 中定义的 VictoriaMetrics provider 配置
        config: "{{ providers.vm }}"
        type: victoriametrics
        with:
          # 查询平均 CPU 使用率
          query: avg(rate(process_cpu_seconds_total))
          queryType: query

  # 基于查询结果执行的操作
  actions:
    - name: create-alert
      provider:
        type: keep
        with:
          # 当 CPU 使用率超过阈值时创建告警
          if: "{{ value.1 }} > 0.0040"
          alert:
            name: "High CPU Usage"
            description: "[Single] CPU usage is high on the VM (created from VM metric)"
            # 根据 CPU 使用率阈值设置严重性
            severity: '{{ value.1 }} > 0.9 ? "critical" : {{ value.1 }} > 0.7 ? "warning" : "info"'
            # 用于过滤和路由的告警标签
            labels:
              environment: production
              app: myapp
              service: api
              team: devops
              owner: alice
```
