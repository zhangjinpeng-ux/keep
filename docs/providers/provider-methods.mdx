Provider 方法是 Provider 在基本 `query` 和 `notify` 能力之外提供的额外功能（[了解更多](/providers/adding-a-new-provider#basics)）。这些方法允许您以更具体的方式与 Provider 的 API 交互，实现更丰富的集成和自动化能力。

## 什么是 Provider 方法？

开发者在每个 Provider 类中使用 `PROVIDER_METHODS` 列表定义 Provider 方法。它们代表可以通过 Provider 的 API 执行的特定操作或查询。这些方法扩展了 Provider 的基本能力，超越了简单的通知和查询。

<Frame>
  <img src="/images/provider-methods-menu.png" />
</Frame>

例如，一个监控服务 Provider 可能暴露以下方法：

- 静音/取消静音告警
- 获取详细追踪信息
- 搜索特定指标
- 修改监控配置

## 使用 Provider 方法

您可以通过以下方式访问 Provider 方法：

- 通过告警操作菜单使用 Keep 的平台界面
- 通过 Keep 的智能 AI 助手（例如，"获取此告警的追踪信息"）
- 通过 Keep 的 API
- 通过 Keep 的工作流

## 添加新的 Provider 方法

要向您的 Provider 添加新方法：

1. 在 Provider 类中定义方法（必须是实例方法）：

```python
def get_traces(self, trace_id: str) -> dict:
    """Get trace details from the provider.

    Args:
        trace_id (str): The ID of the trace to retrieve

    Returns:
        dict: The trace details
    """
    # Implementation
    pass
```

2. 将方法元数据添加到 `PROVIDER_METHODS`：

```python
from keep.providers.models.provider_method import ProviderMethod

PROVIDER_METHODS = [
    ProviderMethod(
        name="Get Traces",
        description="Retrieve trace details",
        func_name="get_traces",
        type="view",  # 'view' 或 'action'
        scopes=["traces:read"],  # 必需的 Provider 权限范围
        category="Observability",  # 可选的方法分组类别
    )
]
```

注意：`func_params` 字段由 Keep 通过反射方法签名自动填充，因此您无需手动定义。

<Warning>
Provider 方法必须是 Provider 类的实例方法（不能是静态方法或类方法）。方法签名会被自动检查，以生成 UI 表单和参数验证。
</Warning>

### 完整示例

以下是一个带有自定义方法的 Provider 完整示例：

```python
class MonitoringProvider(BaseProvider):
    PROVIDER_DISPLAY_NAME = "Monitoring Service"
    
    PROVIDER_METHODS = [
        ProviderMethod(
            name="Mute Alert",
            description="Mute an alert for a specified duration",
            func_name="mute_alert",
            type="action",
            scopes=["alerts:write"],
            category="Alert Management",
        ),
        ProviderMethod(
            name="Get Metrics",
            description="Retrieve metrics for a service",
            func_name="get_metrics",
            type="view",
            scopes=["metrics:read"],
            category="Observability",
        ),
    ]
    
    def mute_alert(self, alert_id: str, duration_minutes: int = 60) -> dict:
        """
        Mute an alert for the specified duration.
        
        Args:
            alert_id: The ID of the alert to mute
            duration_minutes: Duration to mute in minutes (default: 60)
            
        Returns:
            dict: Confirmation of the mute action
        """
        # Implementation here
        response = self._api_call(f"/alerts/{alert_id}/mute", 
                                 {"duration": duration_minutes})
        return {"success": True, "muted_until": response["muted_until"]}
    
    def get_metrics(self, service_name: str, metric_type: str, 
                   time_range: str = "1h") -> list:
        """
        Get metrics for a specific service.
        
        Args:
            service_name: Name of the service
            metric_type: Type of metric (cpu, memory, latency, etc.)
            time_range: Time range for metrics (default: "1h")
            
        Returns:
            list: List of metric data points
        """
        # Implementation here
        return self._query(f"metrics.{metric_type}", 
                          service=service_name, 
                          range=time_range)
```

### 方法类型

- **view**：返回数据用于展示（例如，获取追踪信息、指标）
- **action**：执行一个操作（例如，静音告警、创建工单）

### 参数类型

Provider 方法支持的参数类型：

- `str`：字符串输入字段
- `int`：数字输入字段
- `float`：小数输入字段
- `bool`：布尔复选框
- `datetime`：日期/时间选择器
- `dict`：JSON 对象输入
- `list`：数组/列表输入
- `Literal`：带预定义值的下拉菜单
- `Optional[type]`：指定类型的可选参数

不同参数类型的示例：

```python
from typing import Optional, Literal
from datetime import datetime

def advanced_query(
    self,
    metric_name: str,                                    # Required string
    time_range: Literal["1h", "6h", "24h", "7d"] = "1h", # Dropdown with options
    include_metadata: bool = False,                       # Boolean checkbox
    limit: Optional[int] = None,                          # Optional integer
    start_time: Optional[datetime] = None,                # Optional datetime picker
) -> dict:
    """Query metrics with advanced filtering options."""
    # Implementation
    pass
```

### 自动发现

Keep 会自动检查 Provider 类以：

1. 发现可用的方法
2. 提取参数信息
3. 生成 UI 组件
4. 使 AI 能够理解方法的能力

## 最佳实践

1. **清晰的文档**：为方法提供详细的 docstring
2. **类型提示**：为参数使用 Python 类型提示
3. **错误处理**：返回清晰的错误消息
4. **权限范围**：定义所需的最小权限范围
5. **参数验证**：在执行前验证参数

## 限制

- 目前仅支持同步方法
- 支持的参数类型仅限于基本类型
- 方法必须是 Provider 类的实例方法
- 方法通过反射自动发现
- Keep 基于类型提示验证参数类型
