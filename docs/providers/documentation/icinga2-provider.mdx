# Icinga2 Provider

Icinga2 提供者允许你在 Keep 中接收来自 Icinga2 监控系统的告警。
Icinga2 提供者支持两种接收告警的方法：Webhooks 和 API 轮询。

接收告警的推荐和主要方法是通过 Webhooks。

## 设置

### 前提条件
1. 访问 Icinga2 实例
2. 具有相关权限的 API 用户
3. 具有 webhook 能力的 Keep 实例

### 配置

提供者需要以下配置：

```yaml
authentication:
  host_url: "https://icinga2.example.com"  # 你的 Icinga2 实例 URL
  api_user: "your-api-user"                # Icinga2 API 用户名
  api_password: "your-api-password"        # Icinga2 API 密码
```

### Webhook 配置
要将 Icinga2 配置为通过 webhooks 向 Keep 发送告警：

1. 导航到你的 Icinga2 配置目录
2. 创建或编辑 ```eventcommands.conf``` 文件
3. 添加以下事件命令配置：

```plaintext
object EventCommand "keep-notification" {
  command = [ "curl" ]
  arguments = {
    "-X" = "POST"
    "-H" = "Content-Type: application/json"
    "-H" = "X-API-KEY: ${keep_api_key}"
    "--data" = "{
      \"host\": {
        \"name\": \"$host.name$\",
        \"display_name\": \"$host.display_name$\",
        \"check_command\": \"$host.check_command$\",
        \"acknowledgement\": \"$host.acknowledgement$\",
        \"downtime_depth\": \"$host.downtime_depth$\",
        \"flapping\": \"$host.flapping$\"
      },
      \"service\": {
        \"name\": \"$service.name$\",
        \"display_name\": \"$service.display_name$\",
        \"check_command\": \"$service.check_command$\",
        \"acknowledgement\": \"$service.acknowledgement$\",
        \"downtime_depth\": \"$service.downtime_depth$\",
        \"flapping\": \"$service.flapping$\"
      },
      \"check_result\": {
        \"exit_status\": \"$service.state$\",
        \"state\": \"$service.state_text$\",
        \"output\": \"$service.output$\",
        \"execution_start\": \"$service.last_check$\",
        \"execution_end\": \"$service.last_check$\",
        \"state_type\": \"$service.state_type$\",
        \"attempt\": \"$service.check_attempt$\",
        \"execution_time\": \"$service.execution_time$\",
        \"latency\": \"$service.latency$\"
      }
    }"
    "${keep_webhook_url}" = {
      required = true
    }
  }
}
```
4. 在你的 Icinga2 配置中定义变量：
    - ```keep_api_key```：具有 webhook 角色的 Keep API 密钥
    - ```keep_webhook_url```：你的 Keep Webhook URL
5. 创建一个使用此事件命令的通知规则
6. 重启 Icinga2 以应用更改

### 状态映射

默认情况下，Icinga2 状态自动映射到 Keep 告警严重程度和状态，如下所示：

<TableWrapper>
#### 状态映射
| Icinga2 状态 | Keep 状态 |
|:--------------|:------------|
| OK            | RESOLVED    |
| WARNING       | FIRING      |
| CRITICAL      | FIRING      |
| UNKNOWN       | FIRING      |
| UP            | RESOLVED    |
| DOWN          | FIRING      |

</TableWrapper>
