## 概述

Checkmk 提供者实现了 Keep 与 Checkmk 之间的无缝集成。它允许你通过 webhook 将告警从 Checkmk 发送到 Keep，使你在一个地方管理基础设施和应用程序更加轻松。

## 连接 Checkmk 到 Keep

要将 Checkmk 连接到 Keep，你需要将其配置为来自 Checkmk 的 webhook。按照以下步骤设置集成：

1. Keep webhook 脚本需要安装在 Checkmk 服务器上。

2. 你可以使用以下命令下载 Keep webhook 脚本：

```bash
wget -O webhook-keep.py https://github.com/keephq/keep/blob/main/keep/providers/checkmk_provider/webhook-keep.py?raw=true
```

3. 将下载的脚本复制到 Checkmk 服务器上的以下路径：

如果你使用 Checkmk Docker 容器，则根据你的 Docker 卷映射将其复制到以下路径：

```bash
cp webhook-keep.py /omd/sites/<site_name>/local/share/check_mk/notifications/webhook-keep.py
cd /omd/sites/<site_name>/local/share/check_mk/notifications
```

如果你使用安装在服务器上的 Checkmk，则将其复制到以下路径：

```bash
cp webhook-keep.py ~/local/share/check_mk/notifications/webhook-keep.py
cd ~/local/share/check_mk/notifications
```

4. 使脚本可执行：

```bash
chmod +x webhook-keep.py
```

8. 根据你的要求配置 Rule properties、Contact selections 和 Conditions。

9. 第一个参数是 Keep 的 Webhook URL，即 `https://api.keephq.dev/alerts/event/checkmk`。

10. 第二个参数是 Keep 的 API Key，你可以在 [Keep 设置](https://platform.keephq.dev/settings?selectedTab=users&userSubTab=api-keys)中生成。

11. 点击 Save 保存配置。

12. 现在，当配置的条件满足时，你将开始通过 webhook 从 Checkmk 接收告警到 Keep。
