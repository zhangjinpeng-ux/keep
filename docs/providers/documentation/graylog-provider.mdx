## 概述

**Graylog 提供者** 通过设置 Webhook 连接来支持从 Graylog 接收告警。它允许与 Graylog 无缝集成，通过 Keep 接收有关事件和告警的通知。

<AutoGeneratedSnippet />

## 连接提供者

1. 按照 [Graylog API 访问文档](https://go2docs.graylog.org/current/setting_up_graylog/rest_api_access_tokens.htm?tocpath=Set%20up%20Graylog%7CGet%20Started%20with%20Graylog%7CREST%C2%A0API%7C_____3#CreateanAccessToken) 从你的 Graylog 实例获取 **username** 和 **access token**。
2. 将 **deployment URL** 设置为你的 Graylog 实例的基础 URL（例如，`http://127.0.0.1:9000`）。
3. 确保用户在 Graylog 中具有 **Admin** 角色。

## 功能

**Graylog 提供者** 支持以下关键功能：

- **Webhook 设置**：配置 webhook 以将告警发送到 Keep。
- **告警检索**：根据指定的搜索参数获取和格式化 Graylog 中的告警（最多 10,000 条最近告警）

<Note>
确保 `page` 和 `per_page` 的乘积不超过 10,000。
</Note>

<Note>
Graylog v4.x 的通知 URL 将 api_key 作为查询参数，这是默认行为。
</Note>
