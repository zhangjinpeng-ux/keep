### 主要类方法

- **`setup_webhook(tenant_id, keep_api_url, api_key, setup_alerts)`**
  - `tenant_id (str)`：租户标识符。
  - `keep_api_url (str)`：发送告警数据的 URL。
  - `api_key (str)`：用于认证的 API 密钥。
  - `setup_alerts (bool)`：是否设置告警功能（默认为 True）。

- **`_get_alerts()`**
  - 返回表示告警的 `AlertDto` 对象列表。

## 连接 Provider

要使用 Site24x7 Provider，请使用必要的认证凭据和 Provider 配置进行初始化。确保您的 Zoho 账户凭据（Client ID、Client Secret 和 Refresh Token）在 `Site24x7ProviderAuthConfig` 中正确设置。

## 获取 Refresh Token 的步骤

1. **注册和客户端凭据：**
   - 导航到 [Zoho API Console](https://api-console.zoho.com/)。
   - 使用与您的 Site24x7 账户关联的电子邮件登录或注册。
   - 使用 "Self Client" 选项注册您的应用程序，以获取 Client ID 和 Client Secret。

2. **生成授权令牌：**
   - 前往 Zoho Developer Console，访问您注册的 Self Client。
   - 在 "Generate Code" 选项卡中，输入所需的范围（`Site24x7.Admin.Read, Site24x7.Admin.Create, Site24x7.Operations.Read`）、描述和时间期限。
   - 点击 "Generate" 并复制提供的代码。

3. **生成 Access 和 Refresh Tokens：**
   - 使用授权令牌向 `https://accounts.zoho.com/oauth/v2/token` 发送 POST 请求，以获取 access 和 refresh tokens。

    ```bash
    curl -X POST 'https://accounts.zoho.com/oauth/v2/token' \
         -d 'client_id=your_client_id' \
         -d 'client_secret=your_client_secret' \
         -d 'code=your_grant_token' \
         -d 'grant_type=authorization_code'

   ```

    或者

    ```python
    import requests

    response = requests.post(
        'https://accounts.zoho.com/oauth/v2/token',
        data={
            'client_id': 'your_client_id',
            'client_secret': 'your_client_secret',
            'code': 'your_grant_token',
            'grant_type': 'authorization_code'
        }
    )
    refresh_token = response.json().get('refresh_token')
    ```

## 注意事项

- 您必须使用特定域名的 Zoho Accounts URL 来生成 refresh tokens，否则会收到 `invalid_client` 错误。请参阅 [Zoho Account 的数据中心](https://help.zoho.com/portal/en/kb/accounts/manage-your-zoho-account/articles/data-center-for-zoho-account)。
- 确保在生成授权令牌时包含必要的范围 **Site24x7.Admin.Read, Site24x7.Admin.Create, Site24x7.Operations.Read**，因为它们决定了通过 Provider 可访问的 API 功能。
- Zoho API Console [链接](https://api-console.zoho.com)


## Webhook 集成修改

webhook 集成授予 Keep 在 Site24x7 中访问以下范围的权限：
- `authenticated`
- `valid_tld`

webhook 可以通过 Site24x7 控制台中的 "Alarms" 部分访问。
