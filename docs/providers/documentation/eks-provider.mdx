## 连接提供者
要连接到 Amazon EKS，请按照以下步骤操作：

1. 登录你的 [AWS Console](https://aws.amazon.com/)

2. 创建一个具有 EKS 权限的 IAM 用户：
```bash
aws iam create-user --user-name eks-user
```

3. 附加所需的策略：

```bash
aws iam attach-user-policy --user-name eks-user --policy-arn arn:aws:iam::aws:policy/AmazonEKSClusterPolicy
aws iam attach-user-policy --user-name eks-user --policy-arn arn:aws:iam::aws:policy/AmazonEKSServicePolicy
```

4. 创建访问密钥

```bash
aws iam create-access-key --user-name eks-user
```

你应该得到：

```
{
  "AccessKey": {
    "AccessKeyId": "AKIAXXXXXXXXXXXXXXXX",
    "SecretAccessKey": "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "Status": "Active"
  }
}
```

`AccessKeyId` 是你的 `access_key`，`SecretAccessKey` 是你的 `secret_access_key`。

5. 从 EKS 控制台或使用以下命令记下你的集群名称和区域：

```bash
aws eks list-clusters --region <your-region>
```

## 所需权限
AWS IAM 用户需要以下权限：

1. **eks:DescribeCluster**
2. **eks:ListClusters**

特定操作所需的额外权限：

3. **eks:AccessKubernetesApi** 用于 Pod/Deployment 操作
4. **eks:UpdateCluster** 用于扩缩容操作
