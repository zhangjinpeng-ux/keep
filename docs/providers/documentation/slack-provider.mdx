## 概述

Keep 的 Slack 集成通过允许您向 Slack 发送通知，实现了无缝通信。此集成旨在简化您的工作流程，确保您的团队能够实时获取更新。

### 主要功能

- **直接通知**：利用 Keep 直接向您的 Slack 频道发送消息。
- **灵活配置**：根据 Keep 工作流中的特定触发器轻松配置告警。
- **交互式消息**：使用按钮和输入框等交互式组件增强您的 Slack 消息。
- **可编辑消息**：根据告警状态或其他工作流结果的变化动态更新现有的 Slack 消息，确保您的通知反映最新信息。

## 入门指南

## 认证方法

Keep 的 Slack 集成支持两种主要的认证方法：

- **Webhook URL**：对于简单的通知，使用与您的 Slack 频道关联的 webhook URL。
- **OAuth 2.0**：要获得更集成的体验，请使用 Slack 的 OAuth 2.0 流程授权 Keep。此方法特别适用于需要访问更多 Slack 功能的应用程序。

### 安装

1. **添加到 Slack**：首先点击此页面上的 "Add to Slack" 按钮。您将按照 OAuth 授权流程将 Keep 与您的 Slack 工作区连接。

    <a href="https://slack.com/oauth/v2/authorize?client_id=5336118933622.5881985084595&scope=chat:write&user_scope="><img alt="Add to Slack" height="40" width="139" src="https://platform.slack-edge.com/img/add_to_slack.png" srcSet="https://platform.slack-edge.com/img/add_to_slack.png 1x, https://platform.slack-edge.com/img/add_to_slack@2x.png 2x" /></a>

2. **安装确认**：将 Keep 添加到 Slack 后，您将被重定向到确认页面。此页面将确认安装成功，并提供在 Keep 工作流中充分利用 Slack 通知的后续步骤。

### OAuth 流程

OAuth 流程简化了 Keep 与 Slack 之间的连接，提供了安全的认证和授权方法。

1. **启动 OAuth**：在[平台](https://platform.keephq.dev)中点击 "Slack" Provider。
![OAuth Authorization](/images/slack/slack-oauth.png)
2. **授权 Keep**：按照提示授权 Keep 访问您的 Slack 工作区。

### 设置

1. **创建 Slack App**：如果您还没有，请在 [Slack API Dashboard](https://api.slack.com/apps) 中创建 Slack 应用。
2. **启用 Incoming Webhooks**：在 Slack 应用设置中，启用 Incoming Webhooks 并为您希望发布消息的频道创建 webhook。
3. **使用您的 Webhook URL**：在 Keep 中，使用 webhook URL 向您选择的 Slack 频道发送通知。

## 使用 Keep 的 Slack 集成

安装 Keep 的 Slack 集成后，您就可以通过 Slack 通知增强工作流了。以下是入门方法：

1. **工作流集成**：在 Keep 中，选择您希望添加 Slack 通知的工作流。添加 Slack 通知块，并使用您的消息或告警条件进行配置。

    ![Workflow Configuration](/images/slack/slack-workflow.png)

2. **发送测试通知**：通过配置的工作流发送测试通知，使用 "Run Manually" 链接，确保设置正确。
