## 概述

GCP Monitoring Provider 实现了 Keep 与 GCP Monitoring 之间的无缝集成，允许 GCP Monitoring 的告警通过 webhook 配置直接发送到 Keep。除了告警之外，该提供者现在还支持从 GCP Logging 查询日志条目，使你能够在 Keep 平台内全面查看告警和相关日志。

## 连接 GCP Monitoring 到 Keep

### 通过 Webhook 集成告警

要将 GCP Monitoring 告警连接到 Keep，请在 GCP Monitoring 中将 webhook 配置为通知渠道，并将其链接到所需的告警策略。

### 步骤 1：访问 Notification Channels

登录 Google Cloud Platform 控制台。
导航到 **Monitoring > Alerting > Notification channels**。

### 步骤 2：添加新的 Webhook

在 Webhooks 部分，点击 **ADD NEW**。

<Frame width="100" height="200">

</Frame>

### 步骤 3：配置 Webhook

在 Endpoint URL 字段中，输入 Keep 提供的 webhook URL。

- **Display Name**: keep-gcpmonitoring-webhook-integration
- 启用 **Use HTTP Basic Auth** 并输入以下凭据：
  - **Auth Username**: `api_key`
  - **Auth Password**: `%YOURAPIKEY%`

  <img height="10" src="/images/gcpmonitoring_3.png" />

### 步骤 4：保存 Webhook 配置

- 点击 **Save** 保存 webhook 配置。

### 步骤 5：将 Webhook 关联到告警策略

导航到你希望将通知发送到 Keep 的告警策略。

- 点击 **Edit**。
- 在 "Notifications and name" 下，找到 **Notification Channels** 部分，选择你创建的 `keep-gcpmonitoring-webhook-integration` 渠道。
- 点击 **SAVE POLICY** 保存更改。

## 日志查询集成

GCP Monitoring Provider 还支持从 GCP Logging 查询日志，允许你根据特定过滤器获取日志条目。这对于用相关日志丰富告警数据或在 Keep 中监控特定事件非常有用。

### 认证要求

要启用日志查询，你需要提供一个具有 `logs.viewer` 角色的服务账户 JSON 文件。此服务账户应在 GCP Monitoring Provider 配置的 `authentication` 部分中进行配置。

### 查询日志

提供者的 `query` 函数支持根据资源类型、严重程度或特定关键字等条件过滤日志。你可以使用 `timedelta_in_days` 指定查询日志的时间范围，并使用 `page_size` 控制条目数量。

#### 使用示例

以下是你可能如何使用提供者查询日志条目的示例：

```python
query(filter='resource.type="cloud_run_revision" AND severity="ERROR"', timedelta_in_days=1)
```

这将返回过去一天中与 Cloud Run 修订版本相关的严重级别为 "ERROR" 的日志。

#### 安装后验证

要验证告警和日志，请按照以下步骤操作：

    1.	告警验证：通过在 GCP Monitoring 中触发告警来测试 webhook，并确认它出现在 Keep 中。
    2.	日志查询验证：执行简单的日志查询并验证日志条目是否按预期返回。
