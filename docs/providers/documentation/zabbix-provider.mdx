## 连接 Provider

### API Key

要获取 Zabbix 认证令牌，请按照以下步骤操作，分为 3 个类别（[文档](https://www.zabbix.com/documentation/current/en/manual/web_interface/frontend_sections/users/api_tokens)）：

首先，使用具有特权的用户登录您的 Zabbix 账户（提供的 `zabbix_frontend_url`）。

#### 创建用户角色

1. 导航到 `Users` -> `User Roles` 部分。
2. 在屏幕右上角，点击 `Create user role`
3. 为角色起一个有意义的名称（例如 Keep Role）
4. 在 `User type` 选择框中，选择 `Super Admin`

- 这是因为我们需要的某些范围仅适用于 `Super Admin` 用户类型。[请参阅此处](https://www.zabbix.com/documentation/current/en/manual/api/reference/mediatype/create)

5. 移除所有复选框，除了一个随机的 `Access to UI elements`，这是任何角色都必需的。
6. 在 `API methods` 部分，选择 `Allow list` 并填入以下范围：
- `action.create`
- `action.get`
- `event.acknowledge`
- `mediatype.create`
- `mediatype.get`
- `mediatype.update`
- `problem.get`
- `script.create`
- `script.get`
- `script.update`
- `user.get`
- `user.update`

#### 创建用户

1. 导航到 `Users` -> `Users` 部分。
2. 按照说明添加新用户。为其起一个有意义的用户名（例如 KeepUser）
3. 在 `Permissions` 选项卡中，选择您刚刚创建的角色。
4. 点击 `Add`

#### 创建 API token

1. 导航到 `Users` -> `API tokens` 部分。
2. 在屏幕右上角，点击 `Create API token`
3. 为 API token 起一个有意义的名称（例如 Keep Token）
4. 选择您刚刚创建的用户
5. 取消选中 `Set expiration date and time` 复选框，然后点击 `Add`
6. 复制生成的 API token，保留以备在 Keep 中使用。

## 注意事项

<Note>
  安装 Zabbix webhook 时，Keep 会自动向您的媒体类型中添加一个新的 Keep 类型的媒体类型。

  添加新媒体类型后，Keep 会自动将此媒体类型作为媒体添加到所有现有用户，以便接收来自 Zabbix 的所有告警。
</Note>

## Webhook 集成修改

自动 webhook 集成授予 Keep 在 Zabbix 实例中访问以下范围的权限：
- `mediatype.get`
- `mediatype.update`
- `mediatype.create`
- `user.get`
- `user.update`

您可以在 **Alerts > Media Types** 下查看 webhook 设置。
