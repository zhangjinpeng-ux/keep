## 概述

Dash0 是现代 OpenTelemetry 原生可观察性平台，基于 CNCF 开放标准（如 PromQL、Perses 和 OTLP）构建，具有完全的成本控制。

## 连接 Dash0 到 Keep

要将 Dash0 连接到 Keep，你需要在 Dash0 中创建一个 webhook。

1. 转到 Dash0 仪表板，点击 Organization settings。

<Frame width="100" height="200">
  <img height="10" src="/images/dash0-provider_1.png" />
</Frame>

2. 点击 `Notification Channels`，创建一个类型为 `Webhook` 的新通知通道。

<Frame width="100" height="200">
  <img height="10" src="/images/dash0-provider_2.png" />
</Frame>

3. 为 webhook 命名，输入 [https://api.keephq.dev/alerts/event/dash0](https://api.keephq.dev/alerts/event/dash0) 作为 URL。

4. 按照以下步骤在 Keep 中创建新的 API 密钥。

5. 转到 Keep 仪表板，点击左下角的个人资料图标，然后点击 `Settings`。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-1.png" />
</Frame>

6. 选择 `Users and Access` 标签页，然后选择 `API Keys` 标签页并创建新的 API 密钥。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-2.png" />
</Frame>

7. 命名并选择角色为 `webhook`，然后点击 `Create API Key`。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-3.png" />
</Frame>

8. 复制 API 密钥。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-4.png" />
</Frame>

9. 添加一个新的请求标头，键为 `X-API-KEY`，值为从 Keep 复制的 API 密钥，然后保存 webhook。

<Frame width="100" height="200">
  <img height="10" src="/images/dash0-provider_3.png" />
</Frame>

10. 转到 `Alerting` 下的 `Notifications`，根据需要创建新的通知规则或更改现有通知规则以使用创建的 webhook。

<Frame width="100" height="200">
  <img height="10" src="/images/dash0-provider_4.png" />
</Frame>

11. 转到 `Alerting` 下的 `Checks`，创建新的检查或编辑现有检查以使用创建的通知规则。

<Frame width="100" height="200">
  <img height="10" src="/images/dash0-provider_5.png" />
</Frame>

<Frame width="100" height="200">
  <img height="10" src="/images/dash0-provider_6.png" />
</Frame>

12. 现在你将开始在 Keep 中接收来自 Dash0 的事件。
