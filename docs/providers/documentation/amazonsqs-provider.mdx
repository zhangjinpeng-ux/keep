## 概述

**AmazonSQS 提供者** 支持：
消费 SQS 消息作为告警
向 SQS 队列通知/推送消息

<AutoGeneratedSnippet />

## AmazonSQS 操作的输入

- `message`: str: 通知的正文/消息
- `group_id`: str | None: 仅在队列类型为 FIFO 时必填，普通队列忽略。
- `dedup_id`: str | None: 仅在队列类型为 FIFO 时必填，普通队列忽略。
- **kwargs: dict | None: 你可以传递额外的键值对，它们将作为 MessageAttributes 在通知中发送。

## AmazonSQS 操作的输出
更多详情，请访问 [SQS 文档](https://boto3.amazonaws.com/v1/documentation/api/latest/reference/services/sqs/client/send_message.html#)。
   ```json
   {
        'MD5OfMessageBody': 'string',
        'MD5OfMessageAttributes': 'string',
        'MD5OfMessageSystemAttributes': 'string',
        'MessageId': 'string',
        'SequenceNumber': 'string'
   }
   ```

<Note>
  - 使用 AmazonSQS 操作时，如果队列是 FIFO，则**必须**传递 dedup_id 和 group_id。
  - MessageAttribute 中存在的所有额外字段都作为键值对字典存储在 alert.label 中。
  - 你可以在 SQS 队列消息中传递这些属性，Keep 将提取并使用这些字段用于告警
    - name
    - status: 可选值 'firing' | 'resolved' | 'acknowledged' | 'suppressed' | 'pending'，默认为 'firing'。
    - severity: 可选值 'critical' | 'high' | 'warning' | 'info' | 'low'，默认为 'high'
    - description

</Note>

<Note>
密钥对所需的权限：
1. AmazonSQSFullAccess: 如果你想要通知 + 接收，这是 sqs::read + sqs::write 权限范围。
2. AmazonSQSReadOnlyAccess: 如果你只想接收，这是 sqs::read 权限范围。

你可以在以下路径找到这些：
IAM > Users > [YOUR_USER] > Permission > Add Permissions > Add Permissions > Attach policies directly > 搜索 SQS。

要创建密钥对，请按照以下步骤操作：
1. 在 AWS 控制台中搜索 IAM，按回车。
2. 转到 Users
3. 选择你想要的用户
4. 点击 `Create access key`
5. 选择 `Third party service`，点击 `Next`
6. 添加 `Description Tag`，点击 `Next`
7. 复制/下载密钥对。
</Note>
