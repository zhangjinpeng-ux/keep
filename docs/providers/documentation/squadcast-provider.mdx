## 输入参数

`notify` 函数接受以下参数作为输入：

- `notify_type`（必需）：根据您要创建事件还是备注，取值为 `incident` 或 `notes`。
1. ##### `incident` 的参数
   - `message`（必需）：事件消息。
   - `description`（必需）：事件描述。
   - `tags`（可选）：事件的标签。应为 dict 格式。
   - `priority`（可选）：事件的优先级。
   - `status`（可选）：事件的状态。
   - `event_id`（可选）：event_id 用于解决事件。
   - `additional_json`（可选）：随事件发送的附加 JSON 数据。
2. ##### `notes` 的参数
   - `message`（必需）：备注消息。
   - `incident_id`（必需）：需要创建备注的事件 ID。
   - `attachments`（可选）：备注的附件列表。

更多信息请参阅[文档](https://support.squadcast.com/integrations/incident-webhook-incident-webhook-api)
