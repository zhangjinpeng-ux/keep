## 传统告警与统一告警

Keep 同时支持 Grafana 的传统告警系统和较新的统一告警系统。以下是主要区别：

### 传统告警
- 使用通知渠道进行告警传递
- 在仪表板级别配置
- 使用不同的 API 端点（`/api/alerts` 和 `/api/alert-notifications`）
- 设置更简单，但功能较少
- 告警与仪表板面板紧密耦合

### 统一告警（从 Grafana 9.0 开始为默认）
- 使用告警规则和联络点
- 在 Alerting 部分集中配置
- 使用较新的 `/api/v1/alerts` 端点
- 更强大的功能，包括基于标签的路由
- 支持在单个告警规则中使用多个数据源

<Note>
如果你使用的是 Grafana 8.x 或更早版本，或者在较新版本中显式启用了传统告警，请确保使用传统告警配置来配置 Keep。
</Note>

## 连接提供者

要连接到 Grafana，你需要创建一个 API Token：

1. 登录你的 Grafana 账户。
2. 转到 **Service Accounts** 页面（cmd+k -> service）。
3. 点击 **Add service account** 按钮并为你的服务账户命名。

**替代验证方法（当 Keep 无法从外部访问时）：**

如果 Keep 无法从外部访问且无法创建 webhook，你可以使用以下方法手动验证 Grafana 提供者设置：

1. **在 Grafana 中手动测试告警：**
   - 在 Grafana 中创建手动测试告警。
   - 在 Grafana 中设置一个通常会向 Keep 发送告警的联络点。
   - 触发告警并检查 Grafana 的日志中是否有错误或告警已发送的确认信息。

2. **检查 Grafana 中的日志：**
   - 访问 Grafana 的日志文件或使用 **Explore** 功能查询与告警机制相关的日志。
   - 确保没有与 webhook 集成相关的错误，并且告警被正确处理。

3. **验证集成状态：**
   - 导航到 Grafana 中的 **Alerting** 部分。
   - 确认集成状态显示为活动或正常运行。
   - 监控任何出站 HTTP 请求，以验证 Grafana 正在尝试与 Keep 通信。

4. **网络和连接检查：**
   - 使用网络监控工具确保 Grafana 能够访问 Keep 或配置的任何替代端点以发送告警。

## 服务拓扑

当提供者配置了拓扑数据源 UID 时，Grafana 可以为 Keep 的服务拓扑图贡献数据。Keep 从该数据源查询 Grafana 服务图指标，并使用 `client` 和 `server` 标签构建服务和依赖关系。

<Note>
拓扑图是从 Tempo 收集的追踪数据生成的，并通过 Prometheus 兼容的数据源暴露。

要获取 Datasource UID，请前往：
1. Connections > Data Sources。
2. 点击正在从 Tempo 抓取数据的 Prometheus 实例。URL 格式为 `https://host/connections/datasources/edit/<DATASOURCE_UID>`。
3. 复制该 DATASOURCE_UID 并在安装提供者时使用它。
</Note>

## Webhook 集成修改

Webhook 集成将 Keep 作为联络点添加到 Grafana 实例中。此集成可以在 "Contact Points" 部分下找到。Keep 还获得了以下权限范围：
- `alert.provisioning:read`
- `alert.provisioning:write`
