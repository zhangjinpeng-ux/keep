## 概述

CloudWatch Provider 提供与 AWS CloudWatch 的直接集成，使 Keep 用户能够在 Keep 平台内接收 CloudWatch 告警。此集成集中了监控和告警能力，允许及时响应基础设施或应用程序健康状况的变化。

### 主要特性：

- **Webhook 集成**：支持自动订阅与 CloudWatch 告警关联的 AWS SNS 主题，确保 Keep 收到所有相关告警的通知。
- **支持自定义 SNS 主题**：允许使用已有的 SNS 主题和指定自定义 SNS 主题进行告警通知。
- **广泛的监控范围**：利用 CloudWatch 全面的告警系统来监控应用程序和基础设施的健康状况。
- **灵活的认证**：支持永久和临时 AWS 凭据，以满足各种安全和操作需求。

## 连接提供者

要将 CloudWatch 与 Keep 集成，你需要以下内容：

- 具有访问 CloudWatch 和 SNS 服务权限的 AWS 账户。
- 具有 API 访问权限的已配置 Keep 账户。
- CloudWatch 提供者的适当 AWS IAM 权限。

## 设置集成

<Tip>为了实现无缝设置过程，请确保你的 AWS IAM 角色已正确配置 CloudWatch 和 SNS 访问所需的权限。</Tip>

<AutoGeneratedSnippet />

### 步骤：

1. **配置 AWS IAM 角色**：确保 CloudWatch 提供者使用的 IAM 角色具有 `cloudwatch:DescribeAlarms`、`cloudwatch:PutMetricAlarm`、`sns:ListSubscriptionsByTopic` 和其他相关操作的权限。
2. **指定认证详细信息**：在 Keep 平台中，在 CloudWatch 提供者配置中输入 AWS Access Key、Secret 和 Region 详细信息。
3. **设置 SNS 主题（可选）**：如果使用自定义 SNS 主题，请在提供者配置中指定其 ARN 或名称。Keep 将使用此主题来监听告警通知。
4. **激活提供者**：在 Keep 中完成设置以开始接收 CloudWatch 告警。

## 故障排除

- 确保提供的 AWS 凭据具有正确的权限且未过期。
- 验证 SNS 主题是否正确配置为向 Keep 发送通知。
- 检查 CloudWatch 告警以确保它们处于活动状态并正确配置为在所需条件下触发。

## Webhook 集成修改

CloudWatch 的 Webhook 集成将 Keep 作为订阅者添加到与 CloudWatch 告警关联的 SNS 主题中。此集成允许 Keep 接收 AWS 环境中触发的所有告警的通知。
该集成自动获得 CloudWatch 中的以下权限范围：
- `cloudwatch:DescribeAlarms`
