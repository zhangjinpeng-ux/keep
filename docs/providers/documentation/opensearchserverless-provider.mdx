## 概述

OpenSearch Provider 提供与 **Amazon OpenSearch Serverless** 的原生集成，允许 Keep 用户实时查询、监控和写入文档。这支持运营和安全用例的可观测性和事件驱动告警。

### 主要功能：

- **读写支持**：支持查询和写入文档到 OpenSearch Serverless 集合。
- **AWS IAM 认证**：使用 AWS IAM 凭据（访问密钥/密钥或实例角色）进行认证。

## 连接 Provider

要将 OpenSearch 与 Keep 连接，您需要：

- 一个具有 OpenSearch Serverless (AOSS) 权限的 AWS 账户。
- AOSS 中已配置的集合和索引。
- AWS IAM 凭据（永久或临时）。

## 所需的 AWS IAM 权限（范围）

为正常运行，OpenSearch provider 需要以下 IAM 范围：

### 必需范围

- **`iam:SimulatePrincipalPolicy`**
  - **描述**：用于检查 IAM 身份是否有权访问 AOSS API。
  - **别名**：需要测试以下 3 个范围的访问权限。
  - **是否必需**：是

- **`aoss:APIAccessAll`**
  - **描述**：用于调用 OpenSearch Serverless API。
  - **别名**：调用 serverless API 的访问权限
  - **是否必需**：是

- **`aoss:ListAccessPolicies`**
  - **描述**：需要列出所有数据访问策略。
  - **别名**：策略列表访问
  - **是否必需**：是

- **`aoss:GetAccessPolicy`**
  - **描述**：需要检查每个策略的读写范围。
  - **别名**：策略读取访问
  - **是否必需**：是

- **`aoss:CreateIndex`**
  - **描述**：用于创建索引。
  - **文档**：[AOSS API Docs](https://docs.aws.amazon.com/opensearch-service/latest/developerguide/serverless-genref.html#serverless-operations)
  - **别名**：创建索引
  - **是否必需**：是

- **`aoss:ReadDocument`**
  - **描述**：用于从 OpenSearch 集合中读取文档。
  - **文档**：[AOSS API Docs](https://docs.aws.amazon.com/opensearch-service/latest/developerguide/serverless-genref.html#serverless-operations)
  - **别名**：读取文档
  - **是否必需**：是

- **`aoss:WriteDocument`**
  - **描述**：用于在 OpenSearch 集合中索引或更新文档。
  - **文档**：[AOSS API Docs](https://docs.aws.amazon.com/opensearch-service/latest/developerguide/serverless-genref.html#serverless-operations)
  - **别名**：写入文档
  - **是否必需**：是

<Note>
`iam:SimulatePrincipalPolicy`、`aoss:APIAccessAll`、`aoss:ListAccessPolicies`、`aoss:GetAccessPolicy` 需要从 IAM 控制台添加到 Keep 使用的 IAM 身份中。
另外两个策略是数据访问策略，需要从 AWS serverless 仪表板添加。
请参阅 README 获取逐步设置指南：[README](https://github.com/keep/keep/providers/opensearchserverless_provider\README.md)
</Note>

## 认证配置

要使用 OpenSearch Serverless 进行认证，请提供以下信息：

- **AWS Access Key**（必需）：您的 AWS 访问密钥。
- **AWS Access Key Secret**（必需）：您的 AWS 访问密钥密钥。
- **Region**（必需）：托管 OpenSearch 集合的 AWS 区域。
- **Domain Endpoint**（必需）：AOSS 集合端点的完整域名 URL。


## 设置集成
### 步骤：

1. **分配 IAM 权限**：为您的 IAM 用户/角色授予目标集合上的 `aoss:CreateIndex`、`aoss:ReadDocument` 和 `aoss:WriteDocument` 权限。
2. **配置 Keep Provider**：在 Keep 平台中提供访问密钥、密钥、区域和集合端点。

## 查询 OpenSearch

Keep 支持使用 `_search` 端点的标准 OpenSearch 查询：
- **index**：要查询的 OpenSearch 索引名称。
- **query**：有效的 OpenSearch query DSL 对象。

### 示例

```json
{
  "query": {
    "match_all": {}
  },
  "size": 1
}
```


## 写入 OpenSearch

您可以使用 `_notify` 功能将文档推送到 OpenSearch 集合中。
- **index**：文档应写入的索引名称。
- **document**：表示文档体的 Python 字典。
- **id**：文档的 ID
