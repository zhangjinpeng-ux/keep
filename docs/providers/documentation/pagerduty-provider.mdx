## 描述

PagerDuty Provider 实现了与 PagerDuty 的集成，可以在 Keep 中创建、管理和同步事件和告警。它支持直接 API 密钥认证和 OAuth2，为安全集成提供了更大的灵活性。

## 连接 Provider

将 Keep 连接到 PagerDuty：

- **Routing Key**：用于通过 PagerDuty Events API 发布事件。在 PagerDuty UI 中，这显示为集成密钥。
- **API Key**：用于通过 PagerDuty Incidents API 创建和管理事件。
- **Service Id**（可选）：如果提供，Keep 将在该服务的范围内运行。
- **OAuth2**：令牌管理由 Keep 自动处理。

<Note>
您可以在 PagerDuty Web 应用的 **Services** > **Service Directory** > **Your service** > **Integrations** > **Expand Events API** 下找到 routing key（PagerDuty UI 中的集成密钥），选择您要使用的集成。
您可以在 PagerDuty Web 应用的 **Configuration** > **API Access** 下找到 API 密钥。

routing_key 用于通过 events API 向 PagerDuty 发布事件。
api_key 用于通过 incidents API 创建事件。

</Note>

### 在开源版本中启用 OAuth

如果您想在自托管 Keep 的开源版本中使用 OAuth，可以按照以下步骤操作：

1. 创建一个 PagerDuty 账户
2. 在账户页面中，前往 **Integrations** > **App Registration**
   <Frame>
     <img src="/images/pagerduty-app-registration.png" />
   </Frame>
3. 点击右上角的 **New App** 蓝色按钮
4. 填写必填字段
5. 在 Functionality 部分选择 "OAuth 2.0"，然后点击 **Next**
6. 在 Redirect URL 中，您需要添加 Keep 的 PagerDuty OAuth2 重定向 URL，该 URL 基于您的部署 URL。例如，如果 Keep 部署在 http://localhost:3000，则重定向 URL 为 http://localhost:3000/providers/oauth2/pagerduty
   <Frame>
     <img src="/images/pagerduty-redirect-url.png" />
   </Frame>
7. 在 Authorization 部分，选择 **Scoped OAuth** 并选择以下范围：

- Abilities: Read Access
- Incidents: Read/Write Access
- Services: Read/Write Access
- Webhook Subscriptions: Read/Write Access

8. 点击右下角的 **Register App** 蓝色按钮
9. 从 OAuth 2.0 Client Information 弹窗中复制 **Client ID** 和 **Client Secret**，并在 Keep 后端部署中设置 `PAGERDUTY_CLIENT_ID` 和 `PAGERDUTY_CLIENT_SECRET` 环境变量。
   <Frame>
     <img src="/images/pagerduty-oauth2-credentials.png" />
   </Frame>

## PagerDuty Webhook 集成

默认情况下，当 Keep 将自身安装为 webhook 集成时，它会订阅所有事件事件（"Account Scope"）。

<Frame>
  <img src="/images/pagerduty-account-scope.png" />
</Frame>

如果您希望将 Keep 限制在某些特定服务中，可以通过选择 **Service** 范围并选择您要订阅的服务来实现。

<Frame>
  <img src="/images/pagerduty-service-scope.png" />
</Frame>

在 **Integrations** > **Generic Webhooks (v3)** 下找到此页面。

## 注意事项

该 Provider 使用 events API 或 incidents API 来创建告警或事件。选择使用哪个 API 取决于是否存在 routing_key 或 api_key。

使用 PagerDuty 免费版时，如果试用期已过，可能会导致 Keep 中出现 "pagerduty scopes are invalid" 错误。

## Webhook 集成修改

webhook 集成将 Keep 添加为 PagerDuty 中 "Integrations" API 的目标地址。
这赋予 Keep 在 PagerDuty 中访问以下权限范围的能力：

- `webhook_subscriptions_read`
- `webhook_subscriptions_write`
