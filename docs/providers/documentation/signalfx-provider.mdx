## 概述
SignalFX Provider 通过与 SignalFX Alerting 的 webhook 无缝集成，增强您的监控和告警能力。此集成允许您直接从 SignalFX 接收告警，确保您能够及时了解基础设施内的重要事件和指标。

主要功能：
- Webhook 自动配置：自动将 Keep 配置为 SignalFX 中的 Webhook 集成，订阅所有可用的 SignalFX 检测器和规则，实现全面监控。
- 手动和自动订阅管理：提供灵活性，可以手动添加 Keep 作为新检测器的订阅者，也可以通过从 UI 重新运行"setup webhook"功能进行轻松维护。

<Tip>如需更多信息或帮助，欢迎通过我们的 Slack 社区联系。</Tip>

## 连接 Provider
有三种方式连接 SignalFX：
- Push（手动）- 将 Keep 安装为 Webhook 集成。
- Push（自动配置）- 让 Keep 自动将自身配置为 webhook 集成并订阅您的 SignalFx 检测器。
- Pull - Keep 将从 SignalFx 拉取告警。


<Tip>推荐的安装 SignalFx 方式是通过 Push（自动配置）。使用此方式，您可以享受 Push 方式的优势，包括更多上下文（因为 SignalFx 通过 Webhooks 发送更多上下文）和更实时的告警，同时兼具 Pull 集成的便利性（只需提供凭据，Keep 会处理其余事项）。</Tip>

在接下来的部分中，我们将详细说明每种方式。

### Push（自动配置）
使用此方式：
1. Keep 将自身安装为 Webhook 集成。
2. Keep 遍历所有检测器和规则，并将自身添加为订阅者

<Info>此方式的缺点是您需要具有管理员角色的用户的电子邮件/密码。这是由于 SignalFx 在安装集成方面的限制：您可以在此处了解更多信息 - https://dev.splunk.com/observability/reference/api/integrations/latest#endpoint-create-integration </Info>

## Webhook 集成修改
