## 概述

Grafana Loki 是一个日志聚合系统，旨在存储和查询来自你所有应用程序和基础设施的日志。最简单的入门方式是通过 Grafana Cloud，我们完全可组合的可观察性技术栈。

## 查询 Grafana Loki

Grafana Loki 提供者允许你通过 `query` 和 `query_range` 类型从 Grafana Loki 查询日志。以下是查询可用的参数：

1. `query` 类型：

    - `query`: 要执行的 [LogQL](https://grafana.com/docs/loki/latest/query/) 查询。不使用有效 LogQL 语法的请求将返回错误。
    - `limit`: 返回的最大条目数。默认为 `100`。仅适用于产生流（日志行）响应的查询类型。
    - `time`: 查询的评估时间，以纳秒 Unix 时间戳或其他[支持的格式](https://grafana.com/docs/loki/latest/reference/loki-http-api/#timestamps)表示。默认为当前时间。
    - `direction`: 确定日志的排序顺序。支持的值为 `forward` 或 `backward`。默认为 `backward`。

2. `query_range` 类型：

    - `query`: 要执行的 [LogQL](https://grafana.com/docs/loki/latest/query/) 查询。
    - `limit`: 返回的最大条目数。默认为 `100`。仅适用于产生流（日志行）响应的查询类型。
    - `start`: 查询的开始时间，以纳秒 Unix 时间戳或其他[支持的格式](https://grafana.com/docs/loki/latest/reference/loki-http-api/#timestamps)表示。默认为一小时前。Loki 返回时间戳大于或等于此值的结果。
    - `end`: 查询的结束时间，以纳秒 Unix 时间戳或其他[支持的格式](https://grafana.com/docs/loki/latest/reference/loki-http-api/#timestamps)表示。默认为当前时间。Loki 返回时间戳小于此值的结果。
    - `since`: 用于计算相对于 `end` 的 `start` 的 `duration`。如果 `end` 在未来，则 `start` 计算为当前时间之前的此持续时间。为 `start` 指定的任何值都会取代此参数。
    - `step`: 查询分辨率步长宽度，以 `duration` 格式或浮点秒数表示。`duration` 指 Prometheus 持续时间字符串，格式为 `[0-9]+[smhdwy]`。例如，5m 表示 5 分钟的持续时间。默认值基于 `start` 和 `end` 动态计算。仅适用于产生矩阵响应的查询类型。
    - `interval`: 仅返回在（或大于）指定间隔的条目，可以是 `duration` 格式或浮点秒数。仅适用于产生流响应的查询。不要与 step 混淆，请参阅 [Step versus interval](https://grafana.com/docs/loki/latest/reference/loki-http-api/#step-versus-interval) 下的说明。
    - `direction`: 确定日志的排序顺序。支持的值为 `forward` 或 `backward`。默认为 `backward`。
