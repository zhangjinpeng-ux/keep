## 连接提供者

1. 前往 https://id.atlassian.com/manage-profile/security/api-tokens 创建 API 令牌，生成的令牌应传递给 Jira 认证。
2. 从你的看板 URL 中获取 `host` 和 `board_id`。
3. 从你的项目 > 设置 > 详细信息中获取 `project_key`。
4. `email` 与你的账户邮箱相同。

## 自动转换工作流

Jira 提供者支持在告警状态更改时自动转换工单。这对于保持 Jira 看板与告警状态同步非常有用——例如，在告警解决时自动关闭工单。

### 前提条件

1. 在 Keep 中配置 Jira Cloud 提供者
2. 确保你的 Jira 用户具有 `TRANSITION_ISSUES` 权限
3. 了解你的 Jira 看板名称和所需的转换状态名称

### 工作流 1：告警时创建 Jira 工单

此工作流在告警触发时创建 Jira 工单，但仅当尚未创建工单时才创建。

```yaml
workflow:
  id: jira-create-ticket-on-alert
  name: Create Jira Ticket on Alert
  description: Create Jira ticket when alert fires
  disabled: false
  triggers:
    - type: alert
      cel: status == "firing"
  actions:
    - name: jira-action
      if: "not '{{ alert.ticket_id }}'"
      provider:
        type: jira
        config: "{{ providers.JiraCloud }}"
        with:
          board_name: YOUR_BOARD_NAME  # 将此更改为你的看板名称
          issue_type: Task  # 或 Bug、Story 等
          summary: "{{ alert.name }} - {{ alert.description }}"
          description: |
            "此工单由 Keep 自动创建。

            告警详情：
            {code:json}
            {{ alert }}
            {code}"
          enrich_alert:
            - key: ticket_type
              value: jira
            - key: ticket_id
              value: results.issue.key
            - key: ticket_url
              value: results.ticket_url
```

**关键点：**
- `if: "not '{{ alert.ticket_id }}'"` - 仅当工单尚未存在时才创建
- `enrich_alert` - 将工单 ID、类型和 URL 存储在告警中供以后使用
- 工单在默认状态下创建（通常为 "To Do" 或 "Open"）

### 工作流 2：告警解决时将工单转换为 Done

此工作流更新现有 Jira 工单，并在告警解决时将其转换为 "Done"。

```yaml
workflow:
  id: jira-transition-on-resolved
  name: Transition Jira Ticket to Done
  description: Close Jira ticket when alert is resolved
  disabled: false
  triggers:
    - type: alert
      cel: status == "resolved"
  actions:
    - name: jira-action
      provider:
        type: jira
        config: "{{ providers.JiraCloud }}"
        with:
          issue_id: "{{ alert.ticket_id }}"
          summary: "{{ alert.name }} - {{ alert.description }} (resolved)"
          description: |
            "告警已被 Keep 自动解决。

            解决时间：{{ alert.lastReceived }}

            原始告警详情：
            {code:json}
            {{ alert }}
            {code}"
          transition_to: Done  # 更改为你的工作流状态名称
```

**关键点：**
- 使用富化告警数据中的 `issue_id: "{{ alert.ticket_id }}"`
- `transition_to: Done` - 将工单转换为指定状态
- 不需要 `if` 条件 - 如果告警没有 `ticket_id`，操作将优雅地失败

### 可用的转换名称

常见的 Jira 转换名称（因工作流而异）：
- `Done`
- `Resolved`
- `Closed`
- `In Progress`
- `To Do`
- `Canceled`

**如何查找你的转换名称：**
1. 转到你的 Jira 项目设置
2. 导航到 Workflows
3. 检查工作流中可用的状态
4. 在 `transition_to` 参数中使用确切的状态名称（不区分大小写）

### 错误处理

如果你指定了无效的转换名称，Jira 提供者将返回一条有用的错误消息，列出该工单所有可用的转换：

```
Transition 'Invalid' not found. Available transitions: To Do, In Progress, Done, Closed
```

### 示例：三态工作流

你还可以创建中间转换：

```yaml
# 工作流 3：确认时移至 In Progress
workflow:
  id: jira-transition-in-progress
  name: Transition to In Progress
  description: Move ticket to In Progress when alert is acknowledged
  disabled: false
  triggers:
    - type: alert
      cel: status == "acknowledged"
  actions:
    - name: jira-action
      provider:
        type: jira
        config: "{{ providers.JiraCloud }}"
        with:
          issue_id: "{{ alert.ticket_id }}"
          summary: "{{ alert.name }} - In Progress"
          description: "Alert acknowledged and being worked on."
          transition_to: In Progress
```

### 测试

1. **创建一个触发第一个工作流的告警**
- 验证 Jira 工单是否已创建
- 检查告警是否包含 `ticket_id`、`ticket_type` 和 `ticket_url` 字段

2. **解决告警** 以触发第二个工作流
- 验证现有工单已更新（未创建新工单）
- 检查工单状态是否更改为 "Done"

3. **检查 Keep UI 中的日志** 以查找任何错误或调试信息

### 故障排除

#### 问题：工作流创建了新工单而不是更新

**原因：** 缺少 `issue_id` 参数或告警没有 `ticket_id`。

**解决方案：** 确保第一个工作流使用 `ticket_id` 富化了告警，第二个工作流通过 `issue_id: "{{ alert.ticket_id }}"` 使用它。

#### 问题：转换失败并显示 "Transition 'X' not found"

**原因：** 转换名称与你的 Jira 工作流不匹配。

**解决方案：** 检查错误消息中可用的转换，并相应更新 `transition_to` 参数。

#### 问题：转换时权限被拒绝

**原因：** 你的 Jira 用户没有 `TRANSITION_ISSUES` 权限。

**解决方案：** 在 Jira 项目设置中授予必要的权限。

### 高级功能

#### 配置变量

你可以使用 Keep 的配置变量使工作流更灵活：

```yaml
consts:
  JIRA_BOARD: "ALERTS"
  JIRA_DONE_STATUS: "Done"
  JIRA_ISSUE_TYPE: "Task"

# 然后在工作流中使用：
board_name: "{{ consts.JIRA_BOARD }}"
transition_to: "{{ consts.JIRA_DONE_STATUS }}"
issue_type: "{{ consts.JIRA_ISSUE_TYPE }}"
```

#### 自定义字段

你还可以在创建或更新工单时设置自定义字段：

```yaml
with:
  issue_id: "{{ alert.ticket_id }}"
  summary: "Alert resolved"
  custom_fields:
    customfield_10001: "High"
    customfield_10002: "Production"
  transition_to: Done
```

#### 标签和组件

```yaml
with:
  board_name: YOUR_BOARD_NAME
  summary: "{{ alert.name }}"
  description: "{{ alert.description }}"
  labels:
    - alert
    - automated
    - critical
  components:
    - Monitoring
    - Infrastructure
```
