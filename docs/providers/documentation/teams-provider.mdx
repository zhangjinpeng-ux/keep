## 注意事项

使用 Adaptive Cards（`typeCard="message"`）时：

- `sections` 参数应遵循 [Adaptive Cards 架构](https://adaptivecards.io/explorer/)
- Adaptive Cards 忽略 `themeColor`
- 如果未提供 sections，消息将显示为简单的文本块
- `sections` 和 `attachments` 都可以作为 JSON 字符串或数组提供
- 您可以使用 `mentions` 参数在 Adaptive Cards 中提及用户

### 工作流示例

您也可以在我们的 Keep GitHub 仓库的 [examples](https://github.com/keephq/keep/tree/main/examples/workflows/keep-teams-adaptive-cards.yaml) 文件夹中找到此示例。

```yaml
id: 6bc7c72e-ab3d-4913-84dd-08b9323195ae
description: Teams Adaptive Cards Example
disabled: false
triggers:
  - type: manual
  - filters:
      - key: source
        value: r".*"
    type: alert
consts: {}
name: Keep Teams Adaptive Cards
owners: []
services: []
steps: []
actions:
  - name: teams-action
    provider:
      config: "{{ providers.teams }}"
      type: teams
      with:
        message: ""
        sections: '[{"type": "TextBlock", "text": "{{alert.name}}"}, {"type": "TextBlock", "text": "Tal from Keep"}]'
        typeCard: message
        # Optional: Add mentions to notify specific users
        # mentions: '[{"id": "user@example.com", "name": "User Name"}]'
```

您也可以在我们的 [examples](https://github.com/keephq/keep/tree/main/examples/workflows/keep-teams-adaptive-cards-with-mentions.yaml) 文件夹中找到带有用户提及的示例。

<Note>
    sections 参数是一个遵循 Adaptive Cards 架构的 JSON 字符串，但也可以是对象。
    如果是字符串，它将被解析为 JSON 字符串。
</Note>

### 使用 Sections

```python
provider.notify(
    message="Fallback text",
    typeCard="message",
    sections=[
        {
            "type": "TextBlock",
            "text": "Hello from Adaptive Card!"
        },
        {
            "type": "Image",
            "url": "https://example.com/image.jpg"
        }
    ]
)
```

### 使用自定义 Attachments

```python
provider.notify(
    typeCard="message",
    attachments=[{
        "contentType": "application/vnd.microsoft.card.adaptive",
        "content": {
            "type": "AdaptiveCard",
            "version": "1.2",
            "body": [
                {
                    "type": "TextBlock",
                    "text": "Custom Attachment Example"
                }
            ]
        }
    }]
)
```

### 在 Adaptive Cards 中使用用户提及

您可以使用 `mentions` 参数在 Adaptive Cards 中提及用户。卡片中的文本应包含 `<at>User Name</at>` 格式的提及，并且您需要在 `mentions` 参数中提供用户的 ID 和名称。

Teams 支持三种类型的用户 ID 用于提及：
- Teams User ID（格式：`29:1234...`）
- Microsoft Entra Object ID（格式：`49c4641c-ab91-4248-aebb-6a7de286397b`）
- User Principal Name (UPN)（格式：`user@example.com`）

```python
provider.notify(
    typeCard="message",
    sections=[
        {
            "type": "TextBlock",
            "text": "Hello <at>John Doe</at>, please review this alert!"
        }
    ],
    mentions=[
        {
            "id": "john.doe@example.com",  # Can be UPN, Microsoft Entra Object ID, or Teams User ID
            "name": "John Doe"
        }
    ]
)
```

您也可以在一张卡片中提及多个用户：

```python
provider.notify(
    typeCard="message",
    sections=[
        {
            "type": "TextBlock",
            "text": "Hello <at>John Doe</at> and <at>Jane Smith</at>, please review this alert!"
        }
    ],
    mentions=[
        {
            "id": "john.doe@example.com",
            "name": "John Doe"
        },
        {
            "id": "49c4641c-ab91-4248-aebb-6a7de286397b",  # Microsoft Entra Object ID
            "name": "Jane Smith"
        }
    ]
)
```

在 YAML 工作流中，您可以将 mentions 作为 JSON 字符串提供：

```yaml
actions:
  - name: teams-action
    provider:
      config: "{{ providers.teams }}"
      type: teams
      with:
        typeCard: message
        sections: '[{"type": "TextBlock", "text": "Hello <at>John Doe</at>, please review this alert!"}]'
        mentions: '[{"id": "john.doe@example.com", "name": "John Doe"}]'
```
