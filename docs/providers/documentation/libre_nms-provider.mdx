## 连接 LibreNMS 到 Keep

1. 打开 LibreNMS 仪表板，点击右上角的设置。

<Frame width="100" height="200">
  <img height="10" src="/images/librenms-provider_4.png" />
</Frame>

2. 点击 `Create API access token` 生成新的 API 密钥。

<Frame width="100" height="200">
  <img height="10" src="/images/librenms-provider_5.png" />
</Frame>

3. 为 API 密钥添加描述并点击 `Create API Token`。

<Frame width="100" height="200">
  <img height="10" src="/images/librenms-provider_6.png" />
</Frame>

## Webhooks 集成

1. 打开 LibreNMS 仪表板，在导航栏中打开 `Alerts` 标签页，点击 `Alert Transports`。

<Frame width="100" height="200">
  <img height="10" src="/images/librenms-provider_1.png" />
</Frame>

2. 点击 `Create add transport`，选择 `Transport type` 为 `API`。选择 `API Method` 为 `POST`。

3. 填写 `API URL` 为 [https://api.keephq.dev/alerts/event/libre_nms](https://api.keephq.dev/alerts/event/libre_nms)。

<Frame width="100" height="200">
  <img height="10" src="/images/librenms-provider_2.png" />
</Frame>

4. 复制以下 JSON 并粘贴到 `body` 字段中。

```json
{
  "title": "{{ $title }}",
  "hostname": "{{ $hostname }}",
  "device_id": "{{ $device_id }}",
  "sysDescr": "{{ $sysDescr }}",
  "sysName": "{{ $sysName }}",
  "sysContact": "{{ $sysContact }}",
  "os": "{{ $os }}",
  "type": "{{ $type }}",
  "ip": "{{ $ip }}",
  "display": "{{ $display }}",
  "version": "{{ $version }}",
  "hardware": "{{ $hardware }}",
  "features": "{{ $features }}",
  "serial": "{{ $serial }}",
  "status": "{{ $status }}",
  "status_reason": "{{ $status_reason }}",
  "location": "{{ $location }}",
  "description": "{{ $description }}",
  "notes": "{{ $notes }}",
  "uptime": "{{ $uptime }}",
  "uptime_short": "{{ $uptime_short }}",
  "uptime_long": "{{ $uptime_long }}",
  "elapsed": "{{ $elapsed }}",
  "alerted": "{{ $alerted }}",
  "alert_id": "{{ $alert_id }}",
  "alert_notes": "{{ $alert_notes }}",
  "proc": "{{ $proc }}",
  "rule_id": "{{ $rule_id }}",
  "id": "{{ $id }}",
  "faults": "{{ $faults }}",
  "uid": "{{ $uid }}",
  "severity": "{{ $severity }}",
  "rule": "{{ $rule }}",
  "name": "{{ $name }}",
  "string": "{{ $string }}",
  "timestamp": "{{ $timestamp }}",
  "contacts": "{{ $contacts }}",
  "state": "{{ $state }}",
  "msg": "{{ $msg }}",
  "builder": "{{ $builder }}"
}
```

5. 按照以下步骤在 Keep 中创建新的 API 密钥。

6. 转到 Keep 仪表板，点击左下角的个人资料图标，然后点击 `Settings`。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-1.png" />
</Frame>

7. 选择 `Users and Access` 标签页，然后选择 `API Keys` 标签页并创建新的 API 密钥。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-2.png" />
</Frame>

8. 命名并选择角色为 `webhook`，然后点击 `Create API Key`。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-3.png" />
</Frame>

9. 复制 API 密钥。

<Frame width="100" height="200">
  <img height="10" src="/images/keep-apikey-4.png" />
</Frame>

10. 添加一个键为 `X-API-KEY` 的新标头，在 Keep 中创建新的 API 密钥并将其粘贴为值，然后保存 webhook。

<Frame width="100" height="200">
  <img height="10" src="/images/librenms-provider_2.png" />
</Frame>

11. 保存 webhook。

12. 你可以从 LibreNMS 仪表板的 Devices 标签页中添加设备，并选择你创建的告警传输方式。

<Frame width="100" height="200">
  <img height="10" src="/images/librenms-provider_3.png" />
</Frame>

13. 现在，你将开始在 Keep 中接收告警。
