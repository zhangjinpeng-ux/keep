## 概述

NetBox 是用于建模和记录现代网络的领先解决方案。通过将传统的 IP 地址管理 (IPAM) 和数据中心基础设施管理 (DCIM) 与强大的 API 和扩展相结合，NetBox 为网络自动化提供了理想的"真相来源"。继续阅读，了解为什么全球数千家组织将 NetBox 置于其基础设施的核心。

## 将 NetBox 连接到 Keep

要将 NetBox 连接到 Keep，您需要在 NetBox 中创建一个 webhook。

1. 进入 NetBox 仪表板，点击侧边栏中 `Operations` 部分下的 `Webhooks`。

<Frame width="100" height="200">
  <img height="10" src="/images/netbox-provider_1.png" />
</Frame>

2. 点击 `Add` 按钮添加新的 webhook。

<Frame width="100" height="200">
  <img height="10" src="/images/netbox-provider_2.png" />
</Frame>

3. 输入 [https://api.keephq.dev/alerts/event/netbox](https://api.keephq.dev/alerts/event/netbox) 作为 URL，并选择请求方法为 `POST`。

4. 按照以下步骤在 Keep 中创建新的 API 密钥。

5. 进入 Keep 仪表板，点击左下角的个人资料图标，然后点击 `Settings`。

<Frame width="100" height="200">
  <img height="10" src="/images/netbox-provider_4.png" />
</Frame>

6. 选择 `Users and Access` 选项卡，然后选择 `API Keys` 选项卡，创建一个新的 API 密钥。

<Frame width="100" height="200">
  <img height="10" src="/images/netbox-provider_5.png" />
</Frame>

7. 输入名称，选择角色为 `webhook`，然后点击 `Create API Key`。

<Frame width="100" height="200">
  <img height="10" src="/images/netbox-provider_6.png" />
</Frame>

8. 在 `Additional headers` 字段中，输入 `X-API-KEY` 作为键，步骤 7 中生成的 API 密钥作为值。应如下所示。请参考步骤 3 的截图。

```
X-API-KEY: your-api-key
```

9. 禁用 `SSL verification`（可选）或根据您的需求启用它。

<Frame width="100" height="200">
  <img height="10" src="/images/netbox-provider_7.png" />
</Frame>

10. 点击 `Save` 保存 webhook。

11. 进入侧边栏中 `Operations` 部分下的 `Event Rules`，点击 `Add` 按钮创建新的事件规则。

<Frame width="100" height="200">
  <img height="10" src="/images/netbox-provider_8.png" />
</Frame>

12. 根据您的需求填写必填字段。选择您希望接收事件的 `Object types` 和 `Event types`。

<Frame width="100" height="200">
  <img height="10" src="/images/netbox-provider_9.png" />
</Frame>

13. 在 `Action type` 中选择 `Webhook`，选择步骤 3 中创建的 webhook，然后点击 `Save`。

<Frame width="100" height="200">
  <img height="10" src="/images/netbox-provider_10.png" />
</Frame>

现在，您已成功将 NetBox 连接到 Keep。您将根据创建的事件规则开始在 Keep 中接收事件。
