## 入门指南

要开始使用 Keep GitHub 应用程序，请按照以下简单步骤操作：

1. 注册并登录 **[Keep 平台](https://platform.keephq.dev)**。
2. 通过引导界面或访问 **[此链接](https://github.com/apps/keephq)** 安装 **Keep GitHub 应用程序**。安装过程简单且用户友好。

   <Frame>
     <img src="/images/github-app-install.png" />
   </Frame>

3. 连接您首选的 provider，例如 Datadog，将其关联到 Keep 平台。此步骤使 Keep 能够无缝生成告警并将其部署到您选择的 provider。

   <Frame>
     <img src="/images/connect-provider.png" />
   </Frame>

4. 您现在可以开始使用了！Keep GitHub 应用程序已成功集成到您的 GitHub 工作流中。

## 它是如何工作的？

Keep GitHub 应用程序在后台无缝运行，确保您及时了解仓库中的相关变更。每当 Pull Request 被打开或更新时，应用程序会监控 .keep/ 目录下的文件。

一旦检测到变更，GitHub 应用程序会向 Keep 的 API 智能 AI 层发送 HTTP 请求。AI 层分析变更文件的内容，并结合来自 provider 的上下文（现有告警、示例日志等），基于用户提供的自然语言描述生成告警。AI 驱动的告警生成确保准确性和相关性。

告警生成后，Keep GitHub 应用程序会自动在 Pull Request 中的相应文件上评论该告警。这使您作为用户能够方便地审查和验证生成的告警。

如果生成的告警符合您的要求并准备好部署，您只需在文件上留下评论即可。评论应包含预定义的表情符号之一，例如 🚀 或 🆗（请参阅["使用表情符号部署告警"](#使用表情符号部署告警)部分）。Keep GitHub 应用程序将这些表情符号识别为继续部署过程的命令。

这种直观的工作流简化了告警生成和部署过程，为您提供无缝体验，让您能够专注于项目的核心方面。

## 监控 .keep/ 目录下的文件

Keep GitHub 应用程序主动监控位于仓库父级 `.keep/` 目录下的文件。对这些文件的任何更改或更新都将触发告警生成过程。这使您能够专注于项目的重要方面，同时确保相关变更被及时识别和处理。

## 告警文件结构

`.keep/` 目录下的每个文件代表一个单独的告警。告警文件的结构遵循 YAML 格式。以下是一个告警文件的示例：

```yaml title=alert-example.yaml
# 自然语言的告警文本
alert: |
  Count the error rate (4xx-5xx) this service has in the last 10 minutes.
  Alert when the threshold is above 5% out of total requests.
  Send a Slack message to the #alerts-playground channel and include all the context you have"

# 您之前已连接并希望为此告警生成的 provider
provider: datadog
# 您可以使用此字段覆盖 Keep 托管的 API，让 GitHub 应用程序
# 使用您在本地运行的 API（使用 NGROK URL）
# api_url: https://OVERRIDE-KEEP-MANAGED-API
```

告警文件由以下组件组成：

1. **告警文本**：此部分包含告警的自然语言描述。编写清晰简洁的说明，描述应触发告警的条件或标准。您可以包含任何有助于理解和解决的相关上下文。

2. **Provider**：指定您希望为其生成告警的 provider。这确保告警能够与您现有的监控和通知基础设施无缝集成。在上面的示例中，告警配置为为 Datadog 生成。

3. **API 覆盖**：可选地，您可以包含 api_url 字段来覆盖 Keep 的托管 API。这允许您使用本地托管的 API 进行高级自定义和集成。

## 使用表情符号部署告警

要将告警部署到指定的 provider，您只需在相应文件上使用 🚀 或 🆗 表情符号留下评论即可。Keep GitHub 应用程序将这些表情符号识别为命令，并将相应地启动部署过程。这种简化的方式确保了部署告警时的流畅和直观体验。

例如，通过留下带有 🚀 表情符号的评论，您可以通知 Keep GitHub 应用程序将告警部署到指定的 provider（上述示例中为 Datadog）。

<Frame>
  <img src="/images/first-alert.yaml.png" />
</Frame>

Keep GitHub 应用程序将要么用 👍 标记评论，表示告警已成功部署，要么用 👎 以及另一条带有失败原因的评论，表示告警未部署。

<Info>
  Keep GitHub 应用程序具有重试机制，如果告警未能成功部署到 provider，
  会自动尝试修复。如果部署的告警与最初生成的告警不同，
  Keep GitHub 应用程序将再次评论更新后的版本。
</Info>
